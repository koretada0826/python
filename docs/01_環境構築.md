## 1. ターミナル(=「黒い画面」パソコンへの命令窓口)(10分)

### 1-1. ターミナルって何?

> **「キーボードでパソコンに直接命令する画面」**。
> 普段マウスで「クリック」してることを、文字で打って命令する。

### 1-2. 開き方(Mac)

`Cmd + Space` → 「ターミナル」と入力 → Enter。
**黒い(または白い)画面**が開く。これが「**ターミナル**」。

> **イメージ:** ドラえもんの秘密道具を出す机の引き出しみたいなもの。
> ここに呪文(=コマンド)を打つと、パソコンが動く。

### 1-3. Python が入ってるかチェック

ターミナルに以下を打って Enter:

```
python3 --version
```

→ `Python 3.x.x` が出ればOK(例:`Python 3.11.5`)。

出なかった場合:
1. https://python.org にアクセス
2. 「Download Python」(でかいボタン)を押す
3. インストーラーをダウンロード → ダブルクリック → 「次へ」連打
4. **ターミナルを閉じてもう一度開く**(超重要)
5. もう一回 `python3 --version` を打つ

> **なぜターミナル開き直し?**
> 新しく入れたソフトの場所をターミナルが覚えるのは「次回起動から」。
> 開き直さないと「そんなものない」と言われる。

## 2. VS Code(=「コード専用のノート」)(20分)

### 2-1. VS Code って何?

> **コードを書くために作られた高機能なメモ帳**。
> Microsoft が作ってる **無料**ツール。世界中のエンジニアが使ってる。

普通のメモ帳と違う点:
- **色がつく**(キーワードが見やすい)
- **自動で字下げ**してくれる
- **エラーをその場で教えてくれる**
- **Claude などの拡張機能が入る**

### 2-2. インストール

1. https://code.visualstudio.com/ にアクセス
2. **Download for macOS** を押す(でかいボタン)
3. ダウンロードした zip をダブルクリックで解凍
4. 出てきた `Visual Studio Code.app` を **アプリケーション** フォルダにドラッグ
5. アプリケーションフォルダから起動

> **なぜアプリケーションに移動?**
> 「ダウンロード」フォルダのまま使うと、間違えて消したり Mac が混乱する。
> アプリは **アプリケーションフォルダに置くのがお作法**。

### 2-3. 必須拡張(=「便利機能の追加パーツ」)

VS Code 起動 → 左側のアイコン列の **田んぼマーク**(四角が4つ集まったやつ)をクリック。

入れるもの2つ:
- **Python**(Microsoft 製)→ Python のコードに色をつけて、実行ボタンを出す
- **Japanese Language Pack**(日本語化)→ メニューを日本語にする

検索窓に名前を打って → 「Install」を押す → 入れたら **VS Code を一回閉じて開き直す**。

> **なぜ拡張が必要?**
> VS Code は「素」だとただのメモ帳。Python 拡張を入れて初めて「Python 専用ノート」になる。

## 3. 作業フォルダ(=「自分の作業場」)(10分)

### 3-1. なぜフォルダを作る?

> **コードや資料がバラバラに散らばらないように、専用の引き出しを作る**。
> 部屋を片付けるのと同じ。

### 3-2. 作り方

ターミナルで:

```
mkdir ~/python_lesson    # 「python_lesson」という名前のフォルダを作る
cd ~/python_lesson       # そのフォルダの中に移動する
```

> **読み方:**
> - `mkdir` = make directory(フォルダを作る)
> - `cd` = change directory(フォルダを移動する)
> - `~` = ホームフォルダ(あなたのユーザーフォルダ)の意味

### 3-3. VS Code で開く

VS Code の **ファイル → フォルダーを開く** → `~/python_lesson` を選ぶ。
左に **空っぽのフォルダ**が表示されればOK。これがあなたの作業場。

## 4. Hello, World(=「初めての Python」)(15分)

### 4-1. プログラマーの儀式

> 世界中のプログラマーは、新しい言語を始めるとき必ず「Hello, World!」を画面に出す。
> **これが動いた瞬間が、エンジニア人生のスタート**。

### 4-2. ファイルを作る

VS Code 左側のフォルダ欄で **右クリック → 新しいファイル** → `hello.py` と名前をつける。

> **なぜ `.py`?**
> 拡張子(=ファイル名の最後の `.xxx`)が `.py` だと「これは Python のファイルです」と OS や VS Code に伝わる。

### 4-3. 中身

```python
print("Hello, World!")              # 画面に「Hello, World!」と表示
print(2 + 3)                        # 計算もできる
print("私は" + " " + "Pythonエンジニア")  # 文字をくっつけて表示
```

> **読み方:**
> `print()` は「カッコの中身を画面に表示する命令」。
> Python の中で **一番最初に覚える呪文**。

### 4-4. 保存と実行

- **保存:** `Cmd + S`(超重要。保存しないと反映されない)
- **実行:** 右上の **▶ ボタン**(三角の再生マーク)を押す

下のターミナル欄に:

```
Hello, World!
5
私は Pythonエンジニア
```

→ **🎉 初 Python 動いた**。これでもうあなたはエンジニア候補生。

> **なぜ感動するか?**
> 自分の書いた文字でパソコンが反応した瞬間、世界の見え方が変わる。
> ここから先、全部この延長線上にある。

## 5. GitHub アカウント(=「コードの貯金箱」)(20分)

### 5-1. GitHub って何?

> **書いたコードをインターネット上に保存する場所**。
> パソコンが壊れても、ここに入れておけば失われない。
> エンジニアの「**ポートフォリオ(作品集)**」にもなる。

### 5-2. 作り方

1. https://github.com にアクセス
2. **Sign up** を押す
3. メールアドレス・パスワード・ユーザー名を入力
4. メール認証を済ませる

### 5-3. パソコンに名前を教える

ターミナルで:

```
git config --global user.name "あなたの名前"
git config --global user.email "登録したメアド"
```

> **なぜ?**
> GitHub に「誰がコードを送ってきたか」を伝えるための名札。
> これを設定しないと、後でコードを保存するとき毎回怒られる。

## 7. よくあるエラーと対処(10分)

| エラー・問題 | 原因 | 直し方 |
|---|---|---|
| `python3: command not found` | Python が入ってない or ターミナル再起動忘れ | python.org からインストール → ターミナル開き直し |
| VS Code で実行ボタンが出ない | Python 拡張が入ってない | 拡張機能から「Python」を入れ直し → 再起動 |
| `Permission denied` | フォルダに書き込み権限がない | ホームフォルダ(`~/`)で作業する |
| `git: command not found` | Git が入ってない(Mac は普通入ってる) | `xcode-select --install` を打つ |
| Claude にログインできない | メール認証してない | 受信箱の確認メールから認証 |
| 文字化けする | ターミナルの設定 | ターミナル → 環境設定 → エンコーディング UTF-8 |

### 困った時のテンプレ(必読)

```
今こういう状況:
<何をしようとしてるか>

このエラー出た:
<エラー文をそのままコピペ>

OS は Mac で、ターミナル使ってる。
直して。
```

→ **クロードに丸投げで OK**。9割これで解決する。
