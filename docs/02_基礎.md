## 1. 変数(=「箱」)(10分)

### 1-1. 変数って何?

> **「名前のついた箱」にモノを入れる**だけ。

```python
名前 = "太郎"        # 「名前」という箱に "太郎" という文字を入れる
年齢 = 25            # 「年齢」という箱に 25 という数字を入れる
```

`=` は **「左の箱に右のモノを入れる」** という意味。
**算数の「等しい」じゃない**ので注意!

```python
x = 5
x = x + 1  # 「x の中身に1足したものを、また x の箱に入れる」
print(x)   # → 6
```

### 1-2. 中身の種類(=「型」)

箱に入れるモノには「種類」があって、これを **型** という。

| 型の名前 | 意味 | 例 |
|---|---|---|
| `str`(文字列) | 文字 | `"太郎"`、`"こんにちは"` |
| `int`(整数) | 1, 2, 3... の数 | `25`、`-3`、`0` |
| `float`(浮動小数) | 小数 | `170.5`、`3.14` |
| `bool`(真偽値) | YES / NO | `True`、`False` |

> **なぜ型がある?**
> コンピュータは「数字」と「文字」を別物として扱うから。
> 例えば `"5" + "3"` は `"53"`(文字くっつける)、
> `5 + 3` は `8`(計算する)。**全然違う**。

### 1-3. f-string(超便利)

```python
名前 = "太郎"
年齢 = 25

# 古い書き方(やらなくていい)
print("私は" + str(年齢) + "歳の" + 名前 + "です")

# 新しい書き方(これだけ覚える)
print(f"私は{年齢}歳の{名前}です")
# → 私は25歳の太郎です
```

`f"..."` の **f は format(=書式)の f**。
中に `{変数}` 書くと、その箱の中身が **そこに埋め込まれる**。

> **なぜ便利?**
> 文字と数字を毎回 `str()` で変換する必要がない。
> Claude が書くコードでも **f-string が標準**。

## 2. if(=「もし〜なら」分かれ道)(20分)

### 2-1. 基本

```python
点数 = 75

if 点数 >= 90:
    print("S")
elif 点数 >= 70:
    print("B")
else:
    print("不合格")
```

> **読み方:**
> もし点数が90以上なら "S"。
> そうじゃなくて、もし70以上なら "B"。
> どれにも当てはまらなかったら "不合格"。

### 2-2. インデント(=字下げ)が超重要

```python
if 点数 >= 90:
    print("S")        # ← スペース4つ字下げ(if の中身)
print("おわり")        # ← 字下げなし(if の外、いつも実行)
```

**Python は字下げで「ここから・ここまで」を判定する**。
他の言語みたいに `{}` で囲まない。
**間違えると動かない**。

> **なぜ?**
> Python の作者(グイドさん)が「コードは見やすくあるべき」と決めたから。
> Claude も VS Code も自動で字下げしてくれるので心配無用。

### 2-3. 比較演算子(覚える7つ)

| 記号 | 意味 | 例 |
|---|---|---|
| `==` | **等しい**(=2つで「等しい」) | `年齢 == 25` |
| `!=` | 等しくない | `名前 != "太郎"` |
| `>=` | 以上 | `点数 >= 90` |
| `<=` | 以下 | `点数 <= 30` |
| `and` | **かつ**(両方) | `年齢 >= 18 and 年齢 <= 65` |
| `or` | **または**(どっちか) | `曜日 == "土" or 曜日 == "日"` |
| `not` | ではない | `not 学生` |

> **よくある間違い:** `=` と `==` を間違える。
> - `x = 5` → 箱に入れる
> - `x == 5` → 中身が5かチェック

## 3. for(=「繰り返し」忍者の修行)(20分)

### 3-1. 基本

```python
# 1〜10を表示(range は 1以上、11未満)
for i in range(1, 11):
    print(i)
```

> **読み方:**
> i に 1〜10 を順番に入れて、中身(print)を10回繰り返せ。

### 3-2. リストを回す(よく使う)

```python
果物 = ["りんご", "みかん", "バナナ"]
for f in 果物:
    print(f"今日は {f} を食べた")
```

> **読み方:**
> リストの中身を順番に f に入れて、3回繰り返す。

### 3-3. 合計を出す(実務頻出)

```python
売上 = [1000, 1500, 800, 2000]

# 方法A:for で書く
合計 = 0
for x in 売上:
    合計 = 合計 + x
print(合計)  # 5300

# 方法B:sum() 関数(短い)
print(sum(売上))  # 5300
```

> **どっち使う?**
> 慣れたら B。でも A も読めるようにしておく。

### 3-4. enumerate(番号も欲しい時)

```python
果物 = ["りんご", "みかん", "バナナ"]
for 番号, f in enumerate(果物, start=1):
    print(f"{番号}番目: {f}")
# 1番目: りんご
# 2番目: みかん
# 3番目: バナナ
```

## 4. リスト(=「並べる箱」)(20分)

### 4-1. 基本

```python
果物 = ["りんご", "みかん", "バナナ"]
```

> **イメージ:** 縦長のロッカー。0号室から順番に番号がふられている。

### 4-2. 取り出し方

```python
果物[0]   # "りんご" (1番目=0番から始まる!!)
果物[1]   # "みかん"
果物[-1]  # "バナナ" (最後を -1 で取れる)
果物[-2]  # "みかん" (後ろから2番目)
```

> **なぜ0から?**
> コンピュータの内部の都合。**Python に限らずほぼ全言語が0から**。
> 慣れる以外にない。

### 4-3. 操作

```python
果物.append("ぶどう")  # 末尾に追加 → ["りんご","みかん","バナナ","ぶどう"]
果物.insert(0, "梨")   # 先頭に挿入 → ["梨","りんご",...]
果物.remove("みかん")  # 削除
果物.pop()             # 末尾を取り出す
len(果物)              # 個数を数える
"りんご" in 果物       # True / False(中にあるかチェック)
```

### 4-4. スライス(=「一部分を切り取る」)

```python
数字 = [10, 20, 30, 40, 50]

数字[1:4]   # [20, 30, 40]  (1番目以上、4番目未満)
数字[:3]    # [10, 20, 30]  (最初から3つ)
数字[2:]    # [30, 40, 50]  (2番目以降全部)
数字[::-1]  # [50, 40, 30, 20, 10]  (逆順、超便利)
```

> **これは Python 独特。覚えると一瞬でデータ加工できる**。

## 5. 辞書(=「名札つき箱」)(20分)

### 5-1. 基本

```python
人 = {
    "名前": "太郎",
    "年齢": 25,
    "学生": False
}
```

> **イメージ:** リストは「番号で取り出す」、辞書は「名札で取り出す」。
> 国語辞典が「言葉(=キー)」で「意味(=値)」を引くのと同じ。

### 5-2. 取り出し・追加・削除

```python
人["名前"]              # "太郎"
人["メール"] = "x@y.z"  # 新しい名札で追加
del 人["学生"]          # 削除

# キーが無いとエラーになるので、安全な取り出し方
人.get("住所", "不明")   # キーが無ければ "不明" を返す
```

### 5-3. 全部回す

```python
for key, value in 人.items():
    print(f"{key}: {value}")
# 名前: 太郎
# 年齢: 25
```

> **実務頻出:**
> JSONデータ(API・DBから来るデータ)は **ほぼ全部辞書**。

## 6. 関数(=「ボタン」)(25分)

### 6-1. なぜ関数を作る?

> **同じ処理を何回も書きたくないから**。
> ボタンを1つ作っておけば、何回でも押せる。

### 6-2. 基本

```python
def 税込(税抜, 率=0.10):
    return int(税抜 * (1 + 率))

合計 = 税込(1000)         # 1100
合計2 = 税込(1000, 0.08)  # 1080(率を変える)
```

> **読み方:**
> - `def` = define(定義する)の略
> - `税込` = 関数の名前(=ボタンの名前)
> - `(税抜, 率=0.10)` = 引数(=入れるモノ)。`率=0.10` は「省略したら0.10」
> - `return` = 答えを返す

### 6-3. 引数の渡し方(2種類)

```python
def メール送信(宛先, 件名, 本文):
    print(f"To: {宛先}\nSubject: {件名}\n{本文}")

# 順番で渡す(位置引数)
メール送信("a@b.c", "こんにちは", "元気?")

# 名前で渡す(キーワード引数)← おすすめ
メール送信(宛先="a@b.c", 件名="こんにちは", 本文="元気?")
```

> **キーワード引数のメリット:**
> 順番を覚えなくていい。コード読む人にも分かりやすい。

### 6-4. 戻り値が複数

```python
def 統計(数字リスト):
    return min(数字リスト), max(数字リスト), sum(数字リスト) / len(数字リスト)

最小, 最大, 平均 = 統計([10, 20, 30, 40])
print(最小, 最大, 平均)  # 10 40 25.0
```

## 7. よくあるエラーと対処(10分)

| エラー | 原因 | 直し方 |
|---|---|---|
| `IndentationError` | 字下げが揃ってない | スペース4つに統一 |
| `SyntaxError` | 文法ミス(`:` 忘れなど) | エラー行をクロードに見せる |
| `NameError: name 'x' is not defined` | 箱を作る前に使った | スペル確認、定義順を確認 |
| `TypeError` | 型が違う(文字+数字など) | `str()` `int()` で変換 |
| `KeyError` | 辞書に無いキーを取った | `.get()` を使う |
| `IndexError` | リストの範囲外を取った | `len()` で確認 |

### 困った時のテンプレ(必読)

```
このエラー出た:
<エラー文をそのままコピペ>

コード:
<コードをそのままコピペ>

直して。
```

→ **クロードに丸投げでOK**。
