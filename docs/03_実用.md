## 1. エラーは敵じゃない(=「先生の赤ペン添削」)(20分)

### 1-1. エラーが出る理由

> エラー = **「ここをこう直して」のメッセージ**。
> プログラムが止まったんじゃなくて、**プログラムが助けを求めてる**だけ。

赤い字を見て焦るのは、英語のテストで赤ペンが入ったときと同じ。
**読めば直せる**。読まないから怖い。

### 1-2. エラー文の読み方

```
File "calc.py", line 5, in <module>
    print(10 / x)
ZeroDivisionError: division by zero
```

> **読み方:**
> - `File "calc.py", line 5` → ファイル名と行番号(=どこで起きたか)
> - `ZeroDivisionError` → エラーの種類
> - `division by zero` → 中身(=0で割った)

**この3つ**を読めば9割解決する。

### 1-3. よく出るエラー一覧

| エラー名 | 意味 | 解決 |
|---|---|---|
| `NameError` | 変数名違う(箱が無い) | スペル確認・先に定義したか確認 |
| `TypeError` | 型違う(文字 + 数字とか) | `int()` `str()` で変換 |
| `IndexError` | リスト範囲外を取った | `len()` で長さ確認 |
| `KeyError` | 辞書のキーが無い | `.get()` を使う |
| `IndentationError` | インデント(字下げ)違う | スペース4つに統一 |
| `FileNotFoundError` | ファイルが無い | パス・ファイル名確認 |
| `ZeroDivisionError` | 0で割った | `if x != 0:` でガード |
| `ValueError` | 値の形がおかしい(`int("abc")`など) | 入力チェック |
| `ModuleNotFoundError` | ライブラリが無い | `pip install xxx` で入れる |

### 1-4. try / except でクラッシュ防ぐ(=「保険」)

```python
try:
    数 = int(input("数字: "))      # ここでエラーが起きるかも
    print(10 / 数)
except ValueError:
    print("数字じゃない")           # int 変換失敗の時
except ZeroDivisionError:
    print("0で割れない")            # 0で割った時
except Exception as e:
    print(f"その他のエラー: {e}")    # 念のため全部キャッチ
```

> **読み方:**
> - `try:` の中で **失敗するかもしれない処理**を書く
> - `except XXXError:` で **その種類のエラーが出た時の処理**を書く
> - `except Exception as e:` は「**何でもいいから捕まえる**」最後の砦

### 1-5. なぜ try / except が大事?

> プログラムは **エラーが出たらそこで止まる**。
> 1万件のデータ処理の途中で1件だけおかしくて止まったら、残り9,999件が無駄になる。
> try で囲めば「**1件スキップして次に進む**」ができる。

### 1-6. 解決の魔法(=「Claude に投げる呪文」)

```
このエラー出た。直して:

コード:
<そのままコピペ>

エラー:
<そのままコピペ>
```

## 2. ファイル読み書き(=「ノートに書く・読む」)(30分)

### 2-1. なぜファイル操作?

> プログラムを **閉じても消えない** 場所にデータを残すため。
> 変数(箱)はプログラムが終わると消える。**ファイルは残る**。

### 2-2. テキストファイル(基本中の基本)

```python
# 書き込み(新しく作る or 上書き)
with open("memo.txt", "w", encoding="utf-8") as f:
    f.write("こんにちは\n")     # \n は改行
    f.write("2行目")

# 読み込み(全部一気に)
with open("memo.txt", "r", encoding="utf-8") as f:
    内容 = f.read()
    print(内容)

# 読み込み(行ごとに)
with open("memo.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())  # strip() で改行や余計な空白を削る
```

> **読み方:**
> - `with open(...) as f:` = 「ファイルを開いて、終わったら自動で閉じる」
> - `f` = ファイルを表す変数(別の名前でもOK)
> - `with` を使うと **閉じ忘れがない**(超重要)

### 2-3. モード一覧

| モード | 用途 | 注意 |
|---|---|---|
| `"w"` | 書き込み(上書き) | **既存ファイルは消える** |
| `"r"` | 読み込み | ファイルが無いとエラー |
| `"a"` | 追記(末尾に追加) | 既存内容は残る |
| `"x"` | 新規作成のみ | 既にあるとエラー |

### 2-4. encoding="utf-8" は必須

> **なぜ?**
> 日本語(漢字・ひらがな)を扱うとき、これを書かないと文字化けする。
> Windows だと特に。**おまじないとして毎回書く**。

### 2-5. JSON(=「仕分け箱つき名札」)

```python
import json

data = {"名前": "太郎", "趣味": ["将棋", "読書"], "年齢": 25}

# 書き込み(辞書 → ファイル)
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

# 読み込み(ファイル → 辞書)
with open("data.json", "r", encoding="utf-8") as f:
    data = json.load(f)
print(data["名前"])  # 太郎
```

> **読み方:**
> - `ensure_ascii=False` = 日本語をそのまま保存する(これ書かないと文字化け)
> - `indent=2` = 見やすく字下げして保存

> **JSON ってなぜ大事?**
> Web API・スマホアプリ・データベース、**ほぼ全部 JSON でやり取り**してる。
> 「Python の辞書」とほぼ同じ形なので、Python と相性抜群。

### 2-6. CSV(=「Excel の表をテキストにしたもの」)

```python
import csv

# 書き込み
データ = [
    {"名前": "太郎", "年齢": 25},
    {"名前": "花子", "年齢": 30},
]
with open("data.csv", "w", encoding="utf-8", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["名前", "年齢"])
    writer.writeheader()       # 見出し行を書く
    writer.writerows(データ)    # データを全部書く

# 読み込み
with open("data.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["名前"], row["年齢"])
```

> **`newline=""` は何?**
> Windows で改行が二重になるのを防ぐおまじない。**CSV のときは必ず書く**。

→ ただし **CSV/Excel は次章の pandas がメイン**。ここでは「読み書きできる」程度でOK。

## 3. フォルダ操作(=「ファイルの地図を扱う」)(20分)

### 3-1. pathlib(=「地図アプリ」)

> **ファイルやフォルダの場所を扱う、新しくて便利な道具**。
> 昔は `os` ライブラリ使ってたけど、今は **pathlib が標準**。

```python
from pathlib import Path

p = Path(".")   # "." は「今いるフォルダ」の意味

# 中身一覧
for f in p.iterdir():
    print(f.name)

# 拡張子で絞る(* は「何でも」の意味)
for f in p.glob("*.csv"):
    print(f.name)

# 再帰的(サブフォルダの中まで全部探す)
for f in p.rglob("*.py"):
    print(f.name)

# 存在確認
Path("memo.txt").exists()  # True / False

# フォルダ作成(既にあってもエラーにしない)
Path("新しいフォルダ").mkdir(exist_ok=True)

# パスを組み立てる(/ で繋ぐ)
新パス = Path("data") / "2026" / "04.csv"  # data/2026/04.csv
```

### 3-2. glob(=「ワイルドカード検索」)

| パターン | 意味 |
|---|---|
| `*.csv` | 拡張子が csv のファイル全部 |
| `photo_*.jpg` | `photo_` で始まる jpg |
| `**/*.py` | サブフォルダも含めて py 全部(rglob と同じ) |

> **これが超便利:**
> 「フォルダの中の Excel だけ全部処理したい」みたいな時に **1行で書ける**。

## 5. よくあるエラーと対処

| エラー | 原因 | 直し方 |
|---|---|---|
| `FileNotFoundError` | ファイルが無い、パスが違う | `Path.exists()` で確認、絶対パスを使う |
| `PermissionError` | 書き込み権限がない | フォルダの権限確認、別の場所に保存 |
| `UnicodeDecodeError` | 文字コードが違う | `encoding="utf-8"` を付ける、ダメなら `cp932` |
| `JSONDecodeError` | JSON が壊れてる | ファイルの中身を直接見て確認 |
| ファイルが空っぽになった | `"w"` で開いた | `"r"` で読むだけのときは `"w"` 使わない |
| 改行が二重になる(CSV) | Windows の改行問題 | `newline=""` を付ける |

## 6. 用語まとめ

| 用語 | 一言で |
|---|---|
| エラー | プログラムからの「ここ直して」メッセージ |
| try / except | エラーを捕まえてクラッシュ防ぐ保険 |
| Exception | エラー全部の親分(これで全エラー捕まえられる) |
| with open | ファイルを開いて自動で閉じる書き方 |
| encoding | 文字コード(日本語は utf-8) |
| JSON | データを辞書っぽく保存する形式 |
| CSV | Excel 風の表形式テキスト |
| `json.dump` / `json.load` | 辞書とファイルの相互変換 |
| pathlib | ファイル・フォルダの場所を扱う道具 |
| `Path("...")` | 場所を表すオブジェクト |
| glob | ワイルドカード(`*`)でファイル検索 |
| rglob | サブフォルダ含めて再帰的に検索 |
| `\n` | 改行を表す特殊文字 |
