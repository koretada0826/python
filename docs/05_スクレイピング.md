## 💡 これは何?(小学生でもわかる説明)

****Webサイトから情報を自動で集める**技術**

### なぜ大事?
「**毎日 Amazon の価格チェックして**」みたいな案件が無限にある。

### イメージ
- **スクレイピング** = 「**ネット上のロボットにお使いを頼む**」
- **requests** = 「**ロボットの足**」(サイトを訪問して内容を持ち帰る)
- **BeautifulSoup** = 「**ロボットの目**」(持ち帰ったページから欲しい部分だけ抜き取る)
- **HTML** = 「**Web ページの設計図**」(`<タグ>` で書かれた構造)
- **CSS セレクタ** = 「**部屋の住所表記**」(欲しい部分の場所を指定する書き方)
- **API** = 「**お店の正規の窓口**」(裏口じゃなくて表玄関)
- **User-Agent** = 「**ロボットが名乗る名前**」(ブラウザのフリをする)

ネット上のロボットにお使いを頼んで、結果を Excel に貯める。
**それだけで月10万円**の世界。

## 2. requests の基本(=「ロボットの足」)(20分)

### 2-1. ページを取得する

```python
import requests

r = requests.get("https://example.com")
print(r.status_code)   # 200 = 成功
print(r.text[:500])    # HTML の最初500文字
```

> **読み方:**
> - `requests.get(URL)` = そのページを取ってくる
> - `r.status_code` = 結果の番号(200 なら成功)
> - `r.text` = 取ってきた HTML 全文

### 2-2. ステータスコード(=「お返事の番号」)

| コード | 意味 | どうする? |
|---|---|---|
| **200** | 成功 | OK、続行 |
| 301 / 302 | リダイレクト | URL が変わった(自動追従される) |
| **403** | アクセス拒否 | User-Agent を変える |
| **404** | ページなし | URL を再確認 |
| **429** | アクセスしすぎ | `time.sleep` で待つ |
| 500 | サーバー側エラー | しばらく待ってリトライ |

### 2-3. User-Agent(=「ブラウザのフリ」)

```python
headers = {
    "User-Agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)"
}
r = requests.get(url, headers=headers)
```

> **なぜ必要?**
> 何も指定しないと requests は「**python-requests/2.x**」と名乗る。
> これを見て **「ロボットだ!」と弾く**サイトが多い。
> ブラウザのフリ(=本物のユーザーっぽい名前)をすれば普通に取れる。

### 2-4. パラメータ付きリクエスト

```python
# https://example.com/search?q=Python&page=2
params = {"q": "Python", "page": 2}
r = requests.get("https://example.com/search", params=params)
```

## 3. BeautifulSoup で抽出(=「ロボットの目」)(30分)

### 3-1. HTML の中身を取り出す

```python
from bs4 import BeautifulSoup

html = """
<div class="article">
  <h2 class="title">記事タイトル</h2>
  <p class="body">本文</p>
  <a href="https://example.com">リンク</a>
</div>
"""

soup = BeautifulSoup(html, "lxml")

# タグ名で取得
soup.find("h2").text                          # "記事タイトル"
soup.find("div", class_="article")            # div ごと取る

# CSS セレクタで取得(これがおすすめ)
soup.select_one(".title").text                # "記事タイトル"
soup.select(".article p")                     # p タグ全部(リスト)

# 全リンクを取り出す
for a in soup.find_all("a"):
    print(a.get("href"))
```

> **`find` と `select` の違い:**
> - `find` = 1つだけ取る(古い書き方)
> - `select` = CSS セレクタで取る(モダン)
> - **どちらでも OK だが `select` を覚えるほうがラク**(Web 知識と直結)

### 3-2. CSS セレクタ早見表(=「住所表記」)

| 記号 | 意味 | 例 |
|---|---|---|
| `.class` | クラス名 | `.title` |
| `#id` | ID 名 | `#main` |
| `tag` | タグ名 | `h2` |
| `親 > 子` | 直接の子 | `div > p` |
| `親 子孫` | 全子孫 | `div p` |
| `tag.class` | タグ + クラス | `h2.title` |
| `[属性=値]` | 属性で絞る | `a[href*="github"]` |

### 3-3. HTML の構造を調べる方法(=「設計図を見る」)

> **手順:**
> 1. 欲しい情報があるページを Chrome で開く
> 2. その情報を **右クリック → 検証** を押す
> 3. 開発者ツールが開いて、その要素の HTML が見える
> 4. クラス名や id を確認 → CSS セレクタに使う

**Claude にもこの構造を見せれば、抽出コードを書いてくれる**。

## 4. アクセス間隔(=「マナー」)(5分)

```python
import time

for url in URLs:
    r = requests.get(url, headers=headers)
    # 処理
    time.sleep(1)  # 1秒待つ(必須!)
```

> **なぜ必要?**
> 連続で大量アクセスすると **サーバー攻撃**と見なされる。
> 最悪、**IP ブロック**されたり、**法的トラブル**になる。
> 「**1秒〜数秒の sleep** は人間のマナー」と覚える。

## 5. API を使う(=「正規の窓口」)(30分)

> API = サイトが提供する **公式の窓口**。
> **スクレイピングより 100 倍楽**で、**規約違反になりにくい**。
> あればこっちを使う。

### 5-1. GitHub API の例

```python
import requests

r = requests.get("https://api.github.com/users/octocat")
data = r.json()           # JSON を辞書に変換
print(data["name"])       # "The Octocat"
print(data["followers"])  # フォロワー数
```

> **`r.json()` って何?**
> 返ってきた JSON 文字列を **Python の辞書に自動変換**するメソッド。
> これで普通の辞書として `data["..."]` で値を取れる。

### 5-2. 為替レート API(無料)

```python
r = requests.get("https://open.er-api.com/v6/latest/USD")
data = r.json()
jpy = data["rates"]["JPY"]
print(f"1 USD = {jpy} JPY")
```

### 5-3. API キー付き API

```python
headers = {"Authorization": "Bearer YOUR_API_KEY"}
r = requests.get(url, headers=headers)
```

> **API キーって何?**
> サービス利用するための **会員証番号**。
> サイトに登録して発行してもらう。**コードに直書きせず環境変数に入れる**(後の章で詳しく)。

## 6. JavaScript で動的に読み込まれるサイト(=「待って取る」)(参考)

> 最近のサイトは **JavaScript で後からデータを表示**することが多い。
> その場合 `requests` だと中身が空っぽで取れない。
> その時は **Playwright**(本物のブラウザを Python から操作)。

```
pip install playwright
playwright install chromium
```

```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://example.com")
    page.wait_for_load_state("networkidle")  # 読み込み完了まで待つ
    html = page.content()                     # HTML を取る
    browser.close()
```

→ JS 実行後の HTML が取れる。**慣れたら使う**。

## 9. よくあるエラーと対処

| エラー / 問題 | 原因 | 直し方 |
|---|---|---|
| `403 Forbidden` | User-Agent 無し or ブロック | `User-Agent` ヘッダーを付ける |
| `429 Too Many Requests` | アクセスしすぎ | `time.sleep` を入れる(2-5秒) |
| `404 Not Found` | URL 違う | URL の打ち間違い確認 |
| `ConnectionError` | ネットが繋がらない | Wi-Fi 確認 |
| `find().text` で `AttributeError` | 要素が見つからなかった | `if soup.find(...):` でガード |
| 取りたい部分が空っぽ | JS で後から表示される | Playwright を使う |
| 文字化け | エンコーディング自動判定ミス | `r.encoding = r.apparent_encoding` |
| API キー漏れ | コードに直書き | 環境変数(`.env`)に入れる |
