## 1. API キー取得(=「会員証を発行する」)(20分)

### 1-1. Anthropic(Claude)

1. https://console.anthropic.com に登録
2. 「API Keys」→ 新規発行
3. クレジットカード登録、**$5 チャージで充分**(数千回テストできる)

> **API キーって何?**
> 「**この人が使った分だけ請求します**」を識別するための **長い文字列**。
> `sk-ant-...` みたいな形。**他人に絶対見せない**。

### 1-2. .env でキー安全管理(=「金庫に入れる」)

`.env` ファイル(プロジェクトに置く):
```
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
```

> **なぜ .env に入れる?**
> コードに直書きすると、**GitHub にアップした瞬間世界に公開される**。
> 悪い人に見つかると **数百万円使われる**(本当に起きる)。
> .env に入れて、コードからは `os.getenv("...")` で読む。

### 1-3. .gitignore に追加(必須!)

`.gitignore` ファイル(プロジェクトに置く):
```
.env
__pycache__/
*.pyc
.DS_Store
```

> **`.gitignore` って何?**
> 「**このファイルは GitHub に送らない**」とリストアップする神ファイル。
> .env を絶対これに入れる。**入れ忘れたら一発で破産する**。

→ **.env を GitHub に上げると悪用される**。絶対 .gitignore する。

## 2. Claude API 基本(=「初めての電話」)(20分)

### 2-1. インストール

```
pip install anthropic python-dotenv
```

### 2-2. 最小コード

```python
import os
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()                                      # .env を読み込む
client = Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

response = client.messages.create(
    model="claude-opus-4-7",                        # どのモデルを使うか
    max_tokens=1024,                                # 最大何文字返すか
    messages=[
        {"role": "user", "content": "こんにちは!"}  # 質問
    ]
)

print(response.content[0].text)
```

→ Claude から応答が返ってくる。

> **読み方:**
> - `load_dotenv()` = .env を環境変数として読み込む
> - `client.messages.create(...)` = Claude に質問を投げる
> - `messages=[{"role": "user", "content": "..."}]` = 質問の中身
> - `response.content[0].text` = 返ってきた答えの文字列

### 2-3. role の種類

| role | 意味 |
|---|---|
| `"user"` | あなた(ユーザー)からの発言 |
| `"assistant"` | AI からの返事 |
| `system` パラメータ | AI への役割設定(別枠で渡す) |

### 2-4. system プロンプト(=「役割を与える」)

```python
response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    system="あなたは丁寧で優秀な秘書です。簡潔に答えてください。",
    messages=[{"role": "user", "content": "明日の予定を整理して"}]
)
```

> **system プロンプトの威力:**
> 同じ質問でも、AI の **キャラクター・口調・専門性**を変えられる。
> 「あなたは弁護士」「あなたは小学校の先生」など、用途で使い分ける。

## 5. Streamlit で Web アプリ化(=「画面を作る」)(40分)

### 5-1. Streamlit って何?

> 「**Python だけで Web アプリ作れる**」ライブラリ。
> HTML / CSS / JavaScript **不要**。Python の関数を書くだけで画面ができる。

### 5-2. インストール

```
pip install streamlit
```

### 5-3. 最小チャットボット(`app.py`)

```python
import streamlit as st
import os
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()
client = Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

st.title("AIチャットボット")

# 履歴を session_state(=ページをまたぐ箱)に保存
if "messages" not in st.session_state:
    st.session_state.messages = []

# 過去メッセージを表示
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.write(msg["content"])

# 入力欄
if prompt := st.chat_input("質問してください"):
    # ユーザー発言を追加
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.write(prompt)

    # AI に送る
    r = client.messages.create(
        model="claude-opus-4-7",
        max_tokens=1024,
        messages=st.session_state.messages
    )
    answer = r.content[0].text

    # AI の返事を表示・保存
    st.session_state.messages.append({"role": "assistant", "content": answer})
    with st.chat_message("assistant"):
        st.write(answer)
```

### 5-4. 起動

```
streamlit run app.py
```

→ ブラウザが自動で開いてチャットボットが動く!

> **`session_state` って何?**
> Streamlit で **ページの再読み込みをまたいで残るデータ**を入れる場所。
> ここに会話履歴を入れる = 文脈を覚えてくれる。

### 5-5. Streamlit の便利パーツ

| 命令 | 意味 |
|---|---|
| `st.title("...")` | タイトル表示 |
| `st.write("...")` | 本文表示 |
| `st.text_input("...")` | 入力欄 |
| `st.button("...")` | ボタン |
| `st.sidebar` | サイドバー(モード切替に便利) |
| `st.chat_message` | チャット風吹き出し |
| `st.file_uploader` | ファイルアップロード |
| `st.dataframe(df)` | pandas の表を表示 |

## 6. RAG(=「社内文書 Bot」)— 概念だけ(15分)

> 「**社内の PDF を AI に読ませて、それについて質問できる Bot**」。
> 法律事務所・士業・医療系から **月50万円〜の高単価案件**が取れる。

### 6-1. 仕組み

```
PDF → 細かく分割 → ベクトル化 → DBに保存
   ↓
質問 → ベクトル化 → 関連文書検索 → AIに渡す → 回答
```

> **「ベクトル化」って何?**
> 文章を **数字の列**に変換すること。これにより「**意味が近いか**」を計算できる。
> 「猫」と「ネコ」は別の文字列だけど、ベクトル化すると **似た数字**になる。

## 8. コスト管理(=「お金の話」)(参考)

| モデル | 入力料金 / 1M tokens | 出力料金 / 1M tokens |
|---|---:|---:|
| Claude Opus 4.7 | $15 | $75 |
| Claude Sonnet 4.6 | $3 | $15 |
| Claude Haiku 4.5 | $0.25 | $1.25 |

> **トークンって何?**
> AI が文字を数える単位。日本語は **1文字 ≒ 1〜2トークン** くらい。
> 1M tokens = 100万トークン = だいたい本1冊分。

→ 個人テストは **月 $5〜20 で充分**。大量処理なら Haiku を使う。
**精度が必要なら Opus、ザクザク処理なら Haiku** と使い分ける。

## 9. よくあるエラーと対処

| エラー | 原因 | 直し方 |
|---|---|---|
| `AuthenticationError` | API キー間違い / 未設定 | .env 確認、`load_dotenv()` 呼んだか確認 |
| `RateLimitError` | アクセスしすぎ | `time.sleep` で待つ、Tier を上げる |
| `BadRequestError: max_tokens` | トークン上限超え | `max_tokens` を下げる |
| 残高不足 | クレジット切れ | コンソールから追加チャージ |
| `.env` 読めない | 配置場所違い | プロジェクトルートに置く |
| Streamlit 画面が出ない | ポート 8501 使用中 | 別ポート: `streamlit run app.py --server.port 8502` |
| 文字化け | エンコーディング | `encoding="utf-8"` |
| API キーが GitHub に上がった | `.gitignore` 設定忘れ | **即座にキーを無効化**して再発行 |
