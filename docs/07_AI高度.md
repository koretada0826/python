## 1. RAG とは(=「マニュアルを読ませる仕組み」)(15分)

### 1-1. 名前の由来

**Retrieval Augmented Generation** = 検索(Retrieval)+ 増強(Augmented)+ 生成(Generation)。
**「検索結果で AI を強化する」**という意味。

### 1-2. 全体の流れ

```
[PDF/Word/CSV] → 細かく分割 → ベクトル化(意味を数字に) → DB保存
   ↓
ユーザー質問 → ベクトル化 → DBから関連文書TOP3検索 → AIに渡す → 回答
```

### 1-3. なぜ必要?

> ChatGPT / Claude は **社内情報を知らない**。
> 「うちの就業規則の有給は?」と聞いても「会社によって違います」としか答えられない。
> RAG を使えば:
> - 自社の PDF 50 枚を **事前に読ませて DB に入れる**
> - 「就業規則の有給はどう?」と聞ける
> - AI が **自社情報に基づいて**回答する

→ 法律事務所・士業・医療機関が **月50-200万円で発注**。

### 1-4. なぜ普通の検索じゃダメ?

> Google 検索みたいな「キーワード一致」だと、「**有給**」「**休暇**」「**有休**」「**バケーション**」を別物として扱う。
> ベクトル検索は **意味で似てるか**で測るので、これらを **同じものとして扱える**。

## 4. LangChain Expression Language(LCEL)(=「パイプ繋ぎ」)(30分)

> LangChain の最新書き方。**チェーン処理が楽**。
> Unix の `|`(パイプ)みたいに、処理を **線で繋ぐ**。

```python
from langchain_anthropic import ChatAnthropic
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

llm = ChatAnthropic(model="claude-opus-4-7")
prompt = ChatPromptTemplate.from_messages([
    ("system", "あなたは丁寧な秘書です"),
    ("human", "{question}"),
])

# プロンプト → AI → 出力 をパイプ ( | ) で繋ぐ
chain = prompt | llm | StrOutputParser()
print(chain.invoke({"question": "明日の段取りを教えて"}))
```

> **読み方:**
> - `|`(パイプ) = 「左の出力を右に流す」
> - `prompt | llm | StrOutputParser()` = 「プロンプト作って → AI に流して → 文字列に変換」

## 9. よくあるエラーと対処

| エラー | 原因 | 直し方 |
|---|---|---|
| `chromadb` インストール失敗 | C++ コンパイラ無し | `xcode-select --install`(Mac) |
| 検索が当たらない | チャンクサイズ不適切 | 500-1000で試す、オーバーラップ調整 |
| 回答が古い文書を参照 | 古いベクトルが残ってる | DB を初期化(削除して再インデックス) |
| `RateLimitError` | API 呼びすぎ | バッチ化・キャッシュ・モデル切替(Haiku) |
| Slack Bot が反応しない | Socket Mode 未有効 | Slack App 設定で Socket Mode を ON |
| 文字化け(PDF) | OCR 必要なスキャン PDF | Tesseract で OCR してから RAG |
| 回答が嘘をつく(ハルシネーション) | 該当文書が無い時 AI が想像する | プロンプトで「分からない時は分からないと言う」と指示 |
