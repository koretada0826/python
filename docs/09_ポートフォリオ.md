## 3. スクショ・動画(各30分)

### スクショの撮り方

| 用途 | 方法 |
|---|---|
| 一部範囲 | `Cmd+Shift+4` → 範囲選択 |
| 画面全体 | `Cmd+Shift+3` |
| 特定ウィンドウ | `Cmd+Shift+4 → スペース` |

### GIF動画(動きを見せる)

```bash
brew install --cask kap
```

**Kap** で画面録画 → GIF出力。

または **LICEcap**(Windows/Mac)。

### 撮るべきシーン

1. **入力**(コマンド or ファイルアップロード)
2. **処理中**(進捗バー)
3. **完成**(出力結果)

→ 30秒以内のGIFにすると見やすい。

### 配置

```
01_excel_merger/
└── docs/
    ├── screenshot_input.png
    ├── screenshot_output.png
    └── demo.gif         ← READMEから参照
```

## 4. ⭐ ポートフォリオサイト構築(120分)

GitHub だけでなく**ちゃんとしたWebサイト**を持つと信用2倍。

### 選択肢

| 方法 | 難易度 | 効果 |
|---|---|---|
| **GitHub Pages** | 低 | 無料、HTML静的サイト |
| **Streamlit Cloud** | 低 | 無料、デモが直接動く |
| **Notion 公開ページ** | 超低 | 無料、ノーコード |
| **Vercel + Next.js** | 中 | プロっぽい |
| **WordPress** | 中 | カスタマイズ自由 |

### 推奨:**Streamlit でポートフォリオサイト**

理由: **デモを直接動かせる**(他の人と差別化)

### 実装

クロードに頼む:

```
Streamlit でポートフォリオサイトを作って。

構成:
- トップページ: 自己紹介・スキル・連絡先
- サイドバーで各作品を選べる
- 各作品ページ:
  - 作品名・概要
  - スクショ・GIF
  - 技術スタック
  - 「このアプリを試す」ボタン → 実際に動かせる
  - GitHubリポジトリへのリンク
- 連絡フォーム(SendGrid連携)

5作品分のテンプレ込み。
レスポンシブ(スマホ対応)。
ダークモード切替可能。

requirements.txt も含めて完全なコード書いて。
```

→ クロードがコード書く。デプロイは Streamlit Cloud で無料。

### 公開URL

`https://<ユーザー名>-portfolio.streamlit.app/`

→ ランサーズ提案文に貼る。
