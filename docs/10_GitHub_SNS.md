## 1. Git の超基本(=「作業日記アプリ」)(15分)

### 1-1. Git って何?

> **「コードのバージョン管理」**ツール。
> いつ・誰が・何を変更したか、全部記録される **タイムマシン**。

### 1-2. 用語

| 言葉 | 意味 |
|---|---|
| リポジトリ(repo) | プロジェクトの保管庫 |
| コミット(commit) | 「ここまで」を記録(セーブポイント) |
| プッシュ(push) | ローカル → GitHub に送る |
| プル(pull) | GitHub → ローカルに引っ張る |
| クローン(clone) | GitHub のリポジトリを自分のパソコンに複製 |
| ブランチ(branch) | 作業の枝分かれ |

### 1-3. 基本フロー

```
コード書く
  ↓
git add .            (変更を選ぶ)
  ↓
git commit -m "..."  (記録、セーブポイント作成)
  ↓
git push             (GitHubに送る)
```

> **イメージ:** ゲームのセーブと同じ。
> `add` = どこをセーブするか選ぶ
> `commit` = セーブする
> `push` = クラウドにバックアップ

## 2. 最初のリポジトリ(=「商品棚を立てる」)(30分)

### 2-1. Step 1:GitHub 上で作る

1. https://github.com → 右上「+」→「New repository」
2. 名前: `python-portfolio`
3. **Public**(公開)
4. 「Add README file」のチェックは外す
5. Create repository

### 2-2. Step 2:ローカルで初期化

ターミナル(プロジェクトフォルダで):

```
cd ~/portfolio
git init
```

> **`git init` って何?**
> 「**このフォルダを Git で管理し始める**」という宣言。
> 隠しフォルダ `.git/` ができる(これが Git の脳みそ)。

### 2-3. Step 3:.gitignore を作る

```
# .gitignore
__pycache__/
*.pyc
.env
.DS_Store
output/
*.log
node_modules/
```

→ クロードに「Python 用の .gitignore 作って」で完璧。

> **なぜ `.env` を入れる?**
> Day 6 でやった通り、**API キーが GitHub に流出すると数百万円使われる**。
> `.gitignore` で確実に除外する。

### 2-4. Step 4:コミット

```
git add .
git commit -m "初コミット: ポートフォリオ"
```

### 2-5. Step 5:GitHub に繋ぐ

```
git remote add origin https://github.com/<ユーザー名>/python-portfolio.git
git branch -M main
git push -u origin main
```

→ ブラウザで確認。**作品が世界に公開された**!

### 2-6. 認証エラーが出たら

「Personal Access Token」を作る:

1. GitHub → Settings → Developer settings → Personal access tokens (classic)
2. Generate new token (classic)、scope: `repo`
3. パスワード欄にこのトークン貼る

> **なぜトークン?**
> 2021年からパスワード認証は廃止されてる。
> トークン = **その用途だけに使えるパスワード**。安全。

## 10. よくある問題と対処

| 問題 | 原因 | 直し方 |
|---|---|---|
| `git push` で `Authentication failed` | Personal Access Token 未使用 | トークンをパスワード欄に貼る |
| `.env` を間違えてプッシュした | `.gitignore` 設定漏れ | **即キー無効化** + `git rm --cached .env` |
| README の画像が表示されない | パス間違い | リポジトリ内の相対パスで指定 |
| `git push` で `rejected` | リモートに変更がある | `git pull --rebase` してから push |
| GitHub の草が生えない | コミットしてない / メアド設定間違い | `git config user.email` を GitHub と同じに |
| Twitter フォロワーが増えない | 投稿頻度が低い / 内容が薄い | 1日1投稿 + 数字付きの実例 |
| Note 記事が読まれない | タイトル / SEO 弱い | 「未経験」「〇万円」など検索される語を入れる |
| 直接依頼が来ない | 連絡先が分からない | プロフィールに DM・メール・問い合わせ URL を明記 |
