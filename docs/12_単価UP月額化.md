# Day 12:単価UP・月額化 - 安定収入の柱

## 💡 これは何?(小学生でもわかる説明)

**1件5000円の単発仕事を「月額3万円の会員制」に進化させて、毎月の収入を自動化する日**

### なぜ大事?
毎月新規営業=疲れる。
**月額契約=寝てる間も収入**。これで人生の時間と心の余裕が一気に増える。

### イメージ
- **単発仕事**=「**ラーメン1杯売る**」(毎日新規客が必要)
- **月額契約**=「**会員制ジムの会費**」(自動で毎月引き落とし)
- **領域絞り込み**=「**専門店化**」(なんでも屋より高単価)
- **3本立て営業**=「**3つの井戸**」(1本枯れても水は出る)

### Claude Code に何を頼めばいいか
納品メッセージ・月額提案文・営業DM・スキルシートは全部クロード。
あなたは「**送るタイミングを選ぶ**」だけ。

## 1. 受注後の動き方(=信頼を貯金する)(30分)

### 1-1. なぜ動き方が大事?

> **「動きが速い人」は単価が高い**。
> 同じスキルでも、24時間以内に返事が来る人と3日後に来る人では、**継続率が10倍違う**。

### 1-2. Step 1:24時間以内にヒアリング

クロードに頼む:
```
ランサーズ案件のヒアリングメッセージ書いて。

案件:
<コピペ>

確認したいこと:
- データ件数の想定
- 出力形式の希望
- 例外時の挙動

トーン: 丁寧、相手の負担にならない
```

> **ポイント:** 「確認すること」を**3〜5個に絞る**。
> 10個質問するクライアントは「面倒くさい」と感じて返事しなくなる。

### 1-3. Step 2:中間報告(全体50%時点)

```
〇〇様

進捗のご報告です。

▼ 完了
- ファイル読み込み機能
- データ抽出ロジック

▼ 残り
- エラーハンドリング
- ドキュメント整備

予定通り、明日納品させていただきます。
```

> **なぜ中間報告?**
> クライアントは「**ちゃんと進んでるか不安**」。
> 1通の中間報告で、その不安が消える=信頼度爆上がり。

### 1-4. Step 3:納品(=きれいにラッピング)

ZIPで:
```
project.zip
├── main.py
├── README.md           # 使い方を3行で
├── requirements.txt    # 必要なライブラリ
├── sample_input/       # 入力サンプル
└── sample_output/      # 期待される出力
```

クロードに「納品メッセージ書いて」で完璧な文章作れる。

### よくある間違い

| ミス | 何がダメ | 直し方 |
|---|---|---|
| コードだけ送る | 動かし方がわからん | README必須、3行で書く |
| Python本体が必要だと言わない | クライアントが詰まる | 必要環境を明記 |
| サンプル入出力がない | 動作確認できない | sample_input/output 必ず添付 |

## 2. 評価★5の取り方(=口コミの種を撒く)(20分)

### 2-1. なぜ★5に拘る?

> **★5が3-5件貯まる**と、ランサーズ内の検索順位が上がり、**こちらから営業しなくてもスカウトが来る**ようになる。
> ★4以下は「微妙な人」扱い。**★5以外は意味がない**くらいの気持ちでOK。

### 2-2. ★5を取る5要素

1. **24時間以内に必ず返信**
2. **わかりやすいREADME**(=説明書)
3. **想定外を先回り**(エラー処理しっかり)
4. **修正に快く応じる**(2回まで無料が普通)
5. **最後の一言**:「またのご機会あれば是非お願いします」

→ これで★5率90%。

### 2-3. トラブル対処

| 問題 | 対応 |
|---|---|
| 納期間に合わない | **早めに**(3日前)相談 |
| 検収されない | 14日経過で**自動検収** |
| クレーマー | ランサーズ運営に通報、証拠保存 |
| 仕様が後から変わる | 「追加対応は別途見積り」と最初に明文化 |

## 3. 単価を上げる4段階(=値札を書き換える)(30分)

### 3-1. 段階表(=実績で単価が決まる仕組み)

| ★5の数 | 単価倍率 | 心境 |
|---|---|---|
| 0-3件 | 半額(実績作り) | 「お願いします!」モード |
| 3-10件 | 通常 | 「相場通りで」モード |
| 10-30件 | 1.5倍 | 「私を選ぶならこの値段」モード |
| 30件超 | 2倍以上 | 「断ってもまだ仕事ある」モード |

> **なぜ段階で上げる?**
> ★0で「単価10万円!」と言っても誰も払わない。
> **実績の数=値札の根拠**。実績があるから値段を主張できる。

### 3-2. 単価表を持つ(=メニュー表があるレストラン)

プロフィールに貼る:

```markdown
## 価格表

### 単発
- 簡単な自動化(50行程度): 8,000円〜
- データ集計・グラフ化: 20,000円〜
- スクレイピング(1サイト): 30,000円〜
- AI連携アプリ: 80,000円〜
- RAG検索Bot: 200,000円〜

### 月額契約
- ライト: 30,000円/月(月10時間)
- スタンダード: 50,000円/月(月20時間)
- プレミアム: 100,000円/月(月50時間)
```

> **メニュー表があるメリット:**
> - 値引き要求の盾になる(「これがメニュー表なので…」)
> - 高い案件が向こうから来る(ライトでなくプレミアムを選ぶ客)
> - 自分も「いくらで売ればいいか」迷わない

## 4. ⭐ 月額契約に持ち込む(=会員制ジム化)(40分)

### 4-1. なぜ月額契約が最強?

| 単発(30,000円×5件) | 月額(30,000円×5社) |
|---|---|
| 毎月新規営業必要 | 自動で入る |
| 不安定 | **安定** |
| 営業時間=収入ゼロ | 営業時間=自由時間 |

→ 月額契約**3〜5社作れば月10〜30万円安定**。

> **イメージ:**
> 単発=「**毎日魚を捕る**」
> 月額=「**養殖場を持つ**」
> 養殖場(月額)が3つあれば、釣り(単発)に行かなくても食える。

### 4-2. 提案タイミング(=信頼貯金を使う瞬間)

> **納品時こそ最高のタイミング**。
> クライアントが「お、この人仕事できる」と思ってる瞬間にだけ、月額提案が刺さる。

納品時に投げる:

```
このツール、運用にあたり以下のサポートが必要かと思います。

▼ 月額サポート
- ツール稼働の月次監視
- 軽微な修正・改善
- メールでの相談無制限

月額: 30,000円〜

ご導入企業様の声:
「自動化のおかげで人件費が月10万削減」

ご検討ください。
```

→ 1社2社契約取れれば**月収が安定**する。

### 4-3. 月額提案で外せない3つの要素

1. **何を提供するか**(月10時間サポートなど具体的に)
2. **クライアントのメリット**(人件費削減・トラブル即対応)
3. **金額**(これがないと判断できない)

### よくある失敗

| ミス | なぜダメ | 直し方 |
|---|---|---|
| 納品して即提案 | クライアントの「満足感」が確認できてない | 納品 → 1週間運用後 → 提案 |
| 漠然と「月額やります」 | 何やるか不明で判断できない | 「月10時間まで」など明確化 |
| 値段が高すぎ | 単発の3倍とかは引かれる | 単発単価の半額を月額にする目安 |

## 5. 領域を絞る(=専門店化)(20分)

### 5-1. なぜ絞ると単価が上がる?

> **「Pythonでなんでもやります」**=スーパーの何でも屋
> → 単価10,000円
>
> **「EC事業者向けExcel自動化専門」**=老舗の漬物専門店
> → 単価50,000円

絞ると「**この業界の困りごとを知ってる人**」と認識され、相場が変わる。

### 5-2. おすすめニッチ

| 業界 | 困りごと | 単価 |
|---|---|---|
| **EC(楽天/Amazon/メルカリ)** | 商品データ・在庫管理 | 30,000-100,000円 |
| **不動産** | 物件情報集約 | 30,000-80,000円 |
| **税理士・会計** | 仕訳の自動化 | 50,000-150,000円 |
| **求人・人材** | 応募者データ | 30,000-80,000円 |
| **YouTuber** | チャンネル分析 | 20,000-50,000円 |

→ **どれか1つに絞る**。

### 5-3. 自分に合う領域の選び方

- 興味ある(続くから)
- 知り合いいる(初受注しやすい)
- 過去の職に関連(経験で差別化)

> **3つのうち2つ満たす領域**を選ぶ。

## 6. 営業ルート3本立て(=井戸を3本掘る)(30分)

### 6-1. なぜ3本?

> **1本だと枯れた瞬間に死ぬ**。
> ランサーズが垢BANされたらどうする? Twitter規約変更で投稿制限されたら?
> **3本あれば、1本枯れても残り2本で耐えられる**。

### 6-2. ルート1: 案件サイト(ランサーズ等)→ 即金性高い

毎朝5件応募。慣れたら15分で。
**今すぐ稼ぎたい**ならここ。

### 6-3. ルート2: SNS発信(X/Twitter)→ 半年後から実る

Pythonエンジニア向けプロフィール:
```
🐍 Python × 業務自動化 × AI連携
✅ 月20時間の作業を5秒に
✅ 中小企業のDX支援
✅ Claude/ChatGPT API活用
🔗 ポートフォリオ: <URL>
```

毎日1投稿:
- 「今日作ったツール紹介」
- 「クロード使ってXXを自動化」
- 「数字付きの成果」

→ 半年でフォロワー1,000人 → 直接依頼来始める。

> **コツ:** 数字付き投稿が伸びる。
> 「自動化した」 ❌
> 「月20時間の作業を5秒にした」 ✅

### 6-4. ルート3: 直接DM(中小企業)→ 高単価

```
〇〇株式会社 様

中小企業向けに業務自動化ツールを提供している △△ です。

▼ 弊サービスで実現できること
- 〇〇データの自動取得・更新(月15時間 → 30分)
- レポート自動生成(月10時間 → 0時間)

▼ 過去事例(EC業界)
- 商品データ管理: 月20時間削減

▼ 提供方法
- 初期費用 30,000円〜
- 月額 30,000円〜

無料相談 30分承っております。
```

→ 100社送って5社返信、1社受注のレート。

> **キツいけど高単価**。
> 直接DMで取れる案件は単価10万円〜が普通。

## 7. 確定申告・税金(=年1回の儀式)(15分)

### 7-1. なぜ重要?

> **副業で年20万円超稼いだら確定申告必須**。
> やらないと**追徴課税(税金の倍払い)**。
> 月3万円稼げば年36万円なのですぐ超える。

### 7-2. 副業(年間20万円超)
- 確定申告必要
- **freee** 月980円〜で楽

### 7-3. 経費にできるもの

| 経費 | 例 |
|---|---|
| パソコン代 | 10万以下なら一括、超えたら4年で按分 |
| ネット代 | 自宅の通信費の50%程度 |
| Claude/OpenAI API代 | 100% |
| 本・教材 | Python本・Udemy講座 |
| カフェ代 | 打ち合わせ・作業用 |

→ レシート全部保存。**写真撮ってfreeeに送るだけ**で済む。

### 7-4. 開業届(月10万超えたら)
- 税務署で5分(印鑑とマイナンバーだけ)
- 青色申告で**最大65万円控除**(=実質的に税金20万円ほど安くなる)

### 7-5. 銀行口座
**楽天銀行**を事業用に作る(ランサーズ振込手数料100円)。

> **生活用と分けると、確定申告で「事業の数字」が一目で分かる**。

### 8-1. 月収目標(参考)

| 期間 | 月収 | 案件 |
|---|---:|---|
| 1ヶ月後 | 5,000円〜30,000円 | ランサーズ初受注 |
| 3ヶ月後 | 50,000-150,000円 | ★5件・継続1社 |
| 6ヶ月後 | 150,000-300,000円 | 月額契約3社 |
| **12ヶ月後** | **300,000-500,000円** | 専門領域確立 |

### 8-2. Twitter運用開始

毎日1投稿、3ヶ月続ける → フォロワー500人。
半年で1000人を超えると**個人ブランド**が立ち、案件単価が変わる。

### 8-3. やる気を切らさない仕組み

- 毎週日曜日に「**今週の振り返り日記**」(5行でOK)
- 月末に「**今月の月収・応募数・受注数**」を記録
- 半年に1回、**過去の自分の作品を見返す**

> **自分の成長が可視化されるとモチベが続く**。
