## 1. なぜテストを書く?(15分)

### 趣味コード
- 1度動けばOK
- バグ出たら直す

### 仕事のコード
- **6ヶ月後の自分や他人**が触る
- 追加機能で**既存が壊れない**保証が必要
- 「**テストがない = 信頼できない**」とプロは見る

### テストないコードの末路

```
お客様: "新機能追加して"
あなた: "はい"
→ 既存のA機能が壊れる
→ 客が怒る
→ 信用ゼロ → 月額契約解約
```

→ **テストはあなたの信用を守る盾**。

## 2. pytest 入門(40分)

### インストール

```bash
pip install pytest
```

### 一番簡単なテスト

`utils.py`:
```python
def add(a, b):
    return a + b

def divide(a, b):
    if b == 0:
        raise ValueError("0で割れない")
    return a / b
```

`test_utils.py`(ファイル名 `test_` で始める):
```python
from utils import add, divide
import pytest

def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0

def test_divide():
    assert divide(10, 2) == 5

def test_divide_by_zero():
    with pytest.raises(ValueError):
        divide(10, 0)
```

### 実行

```bash
pytest
# or
pytest -v  # 詳細
pytest test_utils.py::test_add  # 特定だけ
```

→ 緑の `passed` が出れば成功。

## 3. fixture - 共通の前準備(30分)

```python
import pytest

@pytest.fixture
def sample_data():
    return [1, 2, 3, 4, 5]

def test_sum(sample_data):
    assert sum(sample_data) == 15

def test_max(sample_data):
    assert max(sample_data) == 5
```

→ `sample_data` を**両方のテストで使い回せる**。

### よくある fixture

```python
@pytest.fixture
def db_connection():
    """DB接続を準備、テスト後にクローズ"""
    conn = sqlite3.connect(":memory:")
    yield conn
    conn.close()

@pytest.fixture
def temp_csv(tmp_path):
    """一時的なCSVファイル"""
    f = tmp_path / "test.csv"
    f.write_text("a,b,c\n1,2,3")
    return f
```

## 4. パラメータ化テスト(15分)

```python
@pytest.mark.parametrize("a, b, expected", [
    (2, 3, 5),
    (-1, 1, 0),
    (0, 0, 0),
    (100, 200, 300),
])
def test_add(a, b, expected):
    assert add(a, b) == expected
```

→ **1関数で4ケース**を一気にテスト。

## 5. モック - 外部依存を切り離す(30分)

```python
from unittest.mock import patch, MagicMock

def fetch_weather(city):
    import requests
    return requests.get(f"https://api.com/{city}").json()

# テスト(本物のAPIを呼ばない)
@patch("requests.get")
def test_fetch_weather(mock_get):
    mock_get.return_value.json.return_value = {"temp": 25}
    result = fetch_weather("Tokyo")
    assert result["temp"] == 25
```

→ **外部APIなしでテストできる**(高速・確実)。

## 7. 型ヒント(mypy)(30分)

Python は動的型付けだが、**型を明示**するとバグ減る。

```python
def add(a: int, b: int) -> int:
    return a + b

def fetch_user(id: int) -> dict[str, str]:
    return {"name": "太郎", "email": "x@y.z"}

# Optional(値があるかも)
from typing import Optional
def find_user(id: int) -> Optional[dict]:
    if id == 1: return {"name": "太郎"}
    return None
```

### 型チェックツール

```bash
pip install mypy
mypy myproject/
```

→ 型違いを実行前に検出。

### 型ヒントのプロ作法

```python
from typing import Any, Callable, Iterator

# 関数を引数に
def apply(f: Callable[[int], int], x: int) -> int:
    return f(x)

# ジェネレータ
def count(n: int) -> Iterator[int]:
    for i in range(n):
        yield i

# 厳密にしたくない時
def process(data: Any) -> Any:
    ...
```

→ 法人案件は**型ヒントが標準**。書けないと信頼下がる。

### 設定 `pyproject.toml`:

```toml

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B"]
```

## 9. ロギング(20分)

### `print()` は捨てる

❌ `print("エラー")` → 本番で消えない、レベル分けない、ログファイルに残らない
✅ `logging` を使う

```python
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s [%(levelname)s] %(name)s: %(message)s',
    handlers=[
        logging.FileHandler('app.log'),
        logging.StreamHandler()
    ]
)

logger = logging.getLogger(__name__)

logger.debug("詳細")    # 開発時のみ
logger.info("通常")     # 普通の動作
logger.warning("警告")   # 注意
logger.error("エラー")   # エラー(処理は継続)
logger.critical("致命")  # 致命的(停止級)
```

### 構造化ログ(JSONL)

```python
import json
logger.info(json.dumps({
    "event": "user_login",
    "user_id": 123,
    "ip": "1.2.3.4"
}))
```

→ 後から検索・集計しやすい。

## 11. CI でテスト自動化(15分)

`.github/workflows/test.yml`:
```yaml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install -r requirements.txt
      - run: ruff check .
      - run: mypy .
      - run: pytest --cov=.
```

→ push する度に Lint + 型チェック + テスト 自動実行。
