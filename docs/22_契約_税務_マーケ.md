# Day 22:契約・税務・マーケ - 月100万を守るビジネス知識

## 💡 これは何?(小学生でもわかる説明)

**お金まわり全般(契約書・税金・営業)**

### なぜ大事?
技術だけ強くても**搾取される**。これで稼ぎを守る。

> **イメージ:**
> プロボクサーになれてもマネージャーいないと損する。
> マネージャーは「契約交渉(契約書)」「お金の管理(税金)」「試合を取ってくる(営業)」をやる人。
> フリーランスは**自分で兼任**する必要がある。

### たとえ話
- **業務委託契約書**=「仕事の約束を書いた紙」(口約束だと言った言わないで揉める)
- **NDA**=「秘密を守ります、という約束」(機密情報を漏らしたら賠償)
- **インボイス**=「消費税を正しく計算するための領収書ルール」
- **青色申告**=「がんばって帳簿つけたから税金おまけして」という制度
- **経費**=「仕事のために使ったお金」(これを引いた残りに税金がかかる)
- **DM営業**=「自分から「お仕事ください」と知らない人に手紙を送る」

### Claude Code に何を頼めばいいか
この章で覚えるのは「**何ができるか**」と「**何を頼めばいいか**」だけでOK。
契約書ドラフト・営業文・経費区分の判断などは Claude Code に投げる。

## 1. 業務委託契約書(40分)

> **イメージ:** 結婚契約書みたいなもの。
> 「うまくいってる時は読まない」「揉めた時に初めて重要になる」紙。
> でも揉めた時に**書いてないことは戦えない**ので、最初に丁寧に作る。

### 1-1. 必ずチェックする10項目

1. **業務範囲**(何までやる?どこからは追加料金?)
2. **納期**(いつまで?遅れたら?)
3. **報酬**(いくら、いつ振込?)
4. **支払い条件**(月末締め翌月末払い等)
5. **検収条件**(何を満たせば「完了」?)
6. **修正・追加対応**(何回まで無料?)
7. **知的財産権**(成果物の権利は誰のもの?)
8. **秘密保持**(NDA別契約?契約書内?)
9. **損害賠償**(上限額)
10. **解除条件**(どんな時解約できる?)

### 1-2. 危険な条項(=これがあったら必ず修正交渉)

| 条項 | 何が危険? | 修正案 |
|---|---|---|
| 「全ての知的財産権はクライアントに無償譲渡」 | 自分のノウハウ・既存ライブラリまで奪われる | 「本案件で新たに作成した成果物に限る」 |
| 「納品物の一切の不具合の責任を負う」 | 賠償額無限大、人生終わる | 「契約金額を上限とする」 |
| 「無限の修正対応」 | 永遠にタダ働きさせられる | 「契約金額の範囲内で〇回まで」 |
| 「競業避止義務(永久)」 | 別案件取れない | 「契約終了後1年以内、同業界に限る」 |

### 1-3. 修正交渉のテンプレ

> **大事:** 喧嘩腰ではなく「**お互いのリスク回避**」の言い方で。

```
契約書の確認をさせていただきました。
以下の点について修正をお願いできますでしょうか?

1. 7条(知的財産権)
  → 「本件業務に関する成果物の著作権」と限定したい
  → 弊社の汎用ライブラリは含めない

2. 9条(損害賠償)
  → 「契約金額を上限とする」を追記したい

トラブル防止の観点での確認です。
よろしくお願いします。
```

### 1-4. 自分から契約書を出す(直接契約時)

クロードに頼む:

```
業務委託契約書のテンプレを書いて。

仕事内容: Pythonによる業務自動化開発
期間: 単発(納品ベース)
報酬: 〇〇円(税込み)
支払い: 検収後14日以内
納期: 契約日から△日
著作権: 成果物の著作権は乙(私)に帰属、利用権を甲に許諾
損害賠償: 契約金額を上限
解除条件: 双方の合意 or 重大な違反

これでドラフト作成。
```

→ クラウドサインで電子契約締結(印紙不要、無料プランあり)。

> **なぜ電子契約?**
> 紙契約は印紙税がかかる(数千円)。電子なら印紙不要。+ 郵送の手間ゼロ。

## 2. NDA(秘密保持契約)(20分)

法人案件で**ほぼ必ず**出てくる。

> **イメージ:** 「お客さんの家の鍵を預かるとき、誰にも見せません」と書いた約束の紙。
> 漏らしたら高額賠償。

### 2-1. 主要チェック

| 項目 | 確認ポイント |
|---|---|
| **秘密情報の定義** | 何が機密?範囲が広すぎないか |
| **有効期間** | 契約終了後何年?(通常3-5年) |
| **目的外利用禁止** | 当然OKだが範囲を確認 |
| **損害賠償** | 上限額があるか |

### 2-2. 注意すべき条項

| NG条項 | なぜダメ | 修正案 |
|---|---|---|
| 「全ての情報を機密扱い」 | 公知情報まで秘密にされる | 「本契約で授受された情報のうち...」と限定 |
| 「永久に秘密保持」 | 永遠に縛られる | 「3〜5年」に限定 |
| 「違反時 1億円賠償」 | 過大賠償 | 「実損害額」または「契約金額」に修正 |

### 2-3. 自分用 NDA テンプレ

クロードに「フリーランス Python エンジニア用のNDA テンプレを書いて」で生成。

## 3. インボイス制度(2023年〜)(20分)

> **イメージ:** お店のレシートに「**T1234567890123**」という登録番号がないと、買った側が消費税を引けない。
> = 番号がないフリーランスは「消費税分10%値引いて」と言われる。

### 3-1. インボイス登録すべき?

#### YES(登録した方がいい)
- クライアントの大半が**法人(課税事業者)**
- 単価交渉で「インボイスなしだと10%引きで」と言われる
- 売上1,000万円超(自動的に課税事業者)

#### NO(登録しない方がいい)
- クライアントが**個人・免税事業者**中心
- 売上 1,000万円未満で当面増えない

### 3-2. 登録のデメリット

- 消費税納税義務(売上の10%)
- 簡易課税で**みなし仕入率50%**(IT業)→ 実質売上の5%納税

→ クライアントに**法人多い人は登録**、個人相手は様子見。

### 3-3. 確認方法

「**Tから始まる13桁の番号(T1234567890123)を発行**」してもらう。
請求書に必ず明記する。

## 4. 確定申告・節税(40分)

> **イメージ:** 「1年間の収入と経費を税務署に報告して、税金を計算する儀式」。
> 会社員はこれを会社が代わりにやってくれる(=年末調整)。
> フリーランスは**自分で**やる。

### 4-1. 個人事業主の手順

1. **開業届**(税務署、無料、5分)
2. **青色申告承認申請書**(開業届と同時)
3. **freee/マネーフォワード**契約(月980円〜)
4. レシート全部保存(1年間)
5. 翌年2-3月に確定申告

### 4-2. 青色申告の特典

| 特典 | 効果 |
|---|---|
| **65万円控除**(電子申告) | 課税所得が65万円減る=税金10〜20万円減 |
| 赤字を3年繰越可 | 開業初年度赤字でも翌年黒字と相殺 |
| 家族への給与を経費に | 配偶者に給料払って世帯所得分散 |
| 30万円未満の備品を一括経費 | 通常は減価償却が必要だが免除 |

> **なぜ青色?**
> 白色申告は「ドンブリ勘定」、青色は「きちんと帳簿」。
> 帳簿つける手間と引き換えに税優遇。**今は会計ソフトが自動でやる**ので青色一択。

### 4-3. 経費にできるもの(全部入れる)

| 区分 | 例 |
|---|---|
| 通信費 | ネット代、電話代、SaaS月額 |
| 消耗品 | 文房具、PCパーツ |
| 旅費交通費 | 取引先訪問、勉強会 |
| 接待交際費 | クライアント飲食 |
| 図書 | 技術書、講座、Audible |
| 広告宣伝費 | 名刺、ホスティング |
| 研修費 | Udemy、コース、コンサル |
| 減価償却 | PC(10万超は分割) |
| 地代家賃 | 自宅家賃の30-50%(家事按分) |
| 水道光熱費 | 自宅光熱費の30%程度 |
| 租税公課 | 個人事業税、印紙税 |

→ **月収100万 ÷ 経費20万 → 課税所得80万**(節税効果月8万)

> **家事按分とは?**
> 自宅で仕事してると、家賃・光熱費の一部が「事業用」とみなせる。
> 「平日8時間仕事、リビングの3割を仕事スペース」→ 家賃の20-30%を経費。

### 4-4. 節税の上級技

#### 1. 小規模企業共済(年84万円控除)
- 個人事業主向け退職金制度
- 全額所得控除(=その分課税所得が減る)
- いつでも解約可

#### 2. iDeCo(年27.6万円控除)
- 60歳まで引き出せないが**節税効果絶大**
- 年金に上乗せされる

#### 3. ふるさと納税(実質負担2,000円)
- 年収1,000万 → 約17万円分の返礼品

#### 4. 法人化(年商800-1,000万円超で検討)
- 法人税率 23%(個人最大45%より低い)
- 経費の幅広がる(役員報酬・退職金など)
- 退職金積立可能

> **なぜ法人化が節税?**
> 個人は累進課税(稼ぐほど税率UP、最大45%)。
> 法人は一律23%。一定以上稼ぐと法人の方が安い。

## 5. 銀行・口座管理(15分)

### 5-1. 推奨構成

```
[個人用]   楽天銀行・PayPay銀行(普段使い)
[事業用]   GMOあおぞらネット銀行(屋号付き、ランサーズ手数料安)
[貯蓄用]   普通預金(緊急資金 6ヶ月分)
[投資用]   SBI証券・楽天証券(積立NISA、iDeCo)
```

> **なぜ事業用と分ける?**
> 確定申告のとき、個人と事業のお金が混ざってると地獄。
> 最初から分けておけば、事業用口座の動きを見るだけでOK。

### 5-2. 屋号付き口座

開業届出した後、銀行に「**屋号 + 個人名**」で口座作成可。
クライアントから「**◯◯デザイン事務所**」のような屋号で振込もらえる。

→ **対外的な信用UP**(個人名より屋号の方がプロっぽい)。

## 13. ⭐ 月100万維持の鉄則(15分)

1. **時間を売らない**(価値を売る)
2. **月額契約を主軸に**(安定の柱)
3. **専門領域を持つ**(EC/医療/士業など)
4. **AI連携できる**(高単価の証明)
5. **SES + 直接 + SaaS の3階建て**
6. **既存客リピート率 80%超**(新規より既存)
7. **発信を1年継続**(SNS/Note)
8. **法人化の判断を早めに**(節税効果)

> **「時間を売らない」とは?**
> 時給労働=月の上限が決まる(1日10時間が限界)。
> 価値売り=「自動化で月60万円コスト削減した」→ そのうちの月10万を月額でもらう。
> **削減効果と単価が連動**。
