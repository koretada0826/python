## 1. 並行処理の3種類(15分)

| 種類 | 用途 | 実体 |
|---|---|---|
| **マルチスレッド** | I/Oバウンド(ファイル読書) | `threading` |
| **マルチプロセス** | CPUバウンド(画像処理) | `multiprocessing` |
| **非同期(コルーチン)** | ネットワークI/O | `asyncio` |

→ Python の GIL 制約で **CPU処理にはマルチプロセス**、**I/O処理には非同期**が王道。

## 2. asyncio 入門(40分)

```python
import asyncio

async def fetch(url: str) -> str:
    print(f"Start: {url}")
    await asyncio.sleep(1)  # I/O待機を擬似
    print(f"End: {url}")
    return f"Result of {url}"

async def main():
    # 順次(直列)
    r1 = await fetch("a")
    r2 = await fetch("b")
    # 合計 約2秒
    
    # 並行
    results = await asyncio.gather(
        fetch("a"),
        fetch("b"),
        fetch("c"),
    )
    # 合計 約1秒
    print(results)

asyncio.run(main())
```

### キーワード
- `async def`:コルーチン関数の定義
- `await`:非同期処理の完了を待つ
- `asyncio.gather`:並行実行
- `asyncio.run`:イベントループ起動

## 3. aiohttp で 1000並行 HTTPリクエスト(30分)

```bash
pip install aiohttp
```

```python
import asyncio
import aiohttp

async def fetch_url(session, url):
    async with session.get(url) as res:
        return await res.text()

async def main():
    urls = [f"https://api.github.com/repos/python/cpython/issues/{i}" for i in range(1, 51)]
    
    # セマフォで同時実行数制限(レート制御)
    sem = asyncio.Semaphore(10)
    async with aiohttp.ClientSession() as session:
        async def bounded(url):
            async with sem:
                return await fetch_url(session, url)
        
        results = await asyncio.gather(*[bounded(u) for u in urls])
    
    print(f"取得完了: {len(results)}件")

asyncio.run(main())
```

→ 普通の requests なら30秒、これなら**3秒**。

## 4. ThreadPoolExecutor(従来型、簡単)(20分)

```python
from concurrent.futures import ThreadPoolExecutor
import requests

def fetch(url):
    return requests.get(url).status_code

urls = ["https://example.com"] * 50

with ThreadPoolExecutor(max_workers=10) as ex:
    results = list(ex.map(fetch, urls))

print(f"成功: {sum(1 for r in results if r == 200)}")
```

→ 既存コードを最小限の変更で並行化。

## 5. ProcessPoolExecutor(CPU処理)(15分)

```python
from concurrent.futures import ProcessPoolExecutor

def heavy_calc(n):
    return sum(i*i for i in range(n))

with ProcessPoolExecutor(max_workers=4) as ex:
    results = list(ex.map(heavy_calc, [10**6] * 8))
```

→ 4コア並行で**4倍速**(CPU処理)。

## 6. Celery + Redis - バックグラウンドタスク(40分)

「**重い処理をWeb APIから切り離す**」。SaaS の必修。

### インストール

```bash
pip install celery redis
brew install redis
brew services start redis
```

### 構成

```
[FastAPI] → タスク登録 → [Redis Queue] → [Celery Worker(別プロセス)]
              ↑                                    ↓
              └────────── 結果取得 ──────────────┘
```

### 実装

`tasks.py`:
```python
from celery import Celery

app = Celery("tasks", broker="redis://localhost:6379/0", backend="redis://localhost:6379/1")

@app.task
def send_email(to: str, body: str):
    # 重い処理(数十秒)
    print(f"Sending to {to}")
    return f"Sent to {to}"

@app.task
def process_csv(csv_path: str):
    # 大量データ処理
    return "done"
```

`main.py`(FastAPI):
```python
from fastapi import FastAPI
from tasks import send_email

app = FastAPI()

@app.post("/notify")
def notify(email: str):
    task = send_email.delay(email, "Hello")  # 即座にreturn
    return {"task_id": task.id}

@app.get("/result/{task_id}")
def result(task_id: str):
    from tasks import app as celery_app
    task = celery_app.AsyncResult(task_id)
    return {"status": task.state, "result": task.result}
```

### 起動

```bash
# Worker
celery -A tasks worker --loglevel=info

# Beat(定期実行用)
celery -A tasks beat --loglevel=info

# FastAPI
uvicorn main:app
```

→ **メール送信・PDF生成・スクレイピング**を裏で動かす。

## 7. Redis - キャッシュ・キュー・PubSub(30分)

```bash
pip install redis
```

### キャッシュ

```python
import redis
import json

r = redis.Redis(host="localhost", port=6379, decode_responses=True)

# 高速キャッシュ
r.set("user:1", json.dumps({"name": "太郎"}), ex=3600)
data = json.loads(r.get("user:1") or "{}")
```

### キュー(LPUSH / BRPOP)

```python
# Producer
r.lpush("tasks", "task1")
r.lpush("tasks", "task2")

# Consumer(別プロセス)
while True:
    _, task = r.brpop("tasks", timeout=0)
    print(f"Processing: {task}")
```

### PubSub(リアルタイム通知)

```python
# Publisher
r.publish("notifications", json.dumps({"event": "order_created"}))

# Subscriber(別プロセス)
sub = r.pubsub()
sub.subscribe("notifications")
for msg in sub.listen():
    if msg["type"] == "message":
        print(msg["data"])
```

→ チャットアプリ、リアルタイム通知に。
