## 💡 これは何?(小学生でもわかる説明)

- **フロントエンド**=「**お店の客席・メニュー・接客**」(お客から見える部分)

- **フルスタック**=「**料理だけじゃなく接客も配膳もできる料理人**」

## 5. FastAPI と連携(40分)

### 5-1. バックエンド `backend/main.py`

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# CORS 設定(=別ドメインからのアクセス許可)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/api/users")
def list_users():
    return [{"id": 1, "name": "太郎"}, {"id": 2, "name": "花子"}]
```

```bash
uvicorn main:app --reload
```

> **CORS とは?**
> Cross-Origin Resource Sharing。
> ブラウザは「**別ドメインへのアクセス**」をデフォで拒否する。
> CORS 設定で許可リストに入れる。

### 5-2. フロント側

`src/app/users/page.tsx`:
```tsx
async function UsersPage() {
  const users = await fetch("http://localhost:8000/api/users", {
    cache: "no-store"  // 毎回フェッチ(キャッシュしない)

    <ul className="space-y-2 p-8">
```

## 8. JWT で FastAPI と連携(20分)

> **JWT とは?**
> JSON Web Token。「**ログイン済み証明書**」のような暗号化文字列。
> Next 側で発行、FastAPI 側で検証。

```python
# FastAPI 側
from jose import jwt
SECRET = "shared-secret"

@app.middleware("http")
async def verify_jwt(request, call_next):
    token = request.headers.get("Authorization", "").replace("Bearer ", "")
    if token:
        try:
            payload = jwt.decode(token, SECRET, algorithms=["HS256"])
            request.state.user_id = payload["sub"]
        except: pass
    return await call_next(request)
```

```ts
// Next 側でトークンを送る
const session = await auth();
const res = await fetch("http://localhost:8000/api/me", {
  headers: { Authorization: `Bearer ${session?.token}` }
});
```

## 11. WebSocket / Server-Sent Events(20分)

### 11-1. SSE(片方向の継続送信)

> **SSE とは?**
> Server-Sent Events。サーバー→ブラウザの一方通行ストリーム。
> WebSocket より軽量。AIチャットの「逐次表示」で必須。

`src/app/api/stream/route.ts`:
```ts
export async function GET() {
  const stream = new ReadableStream({
    async start(controller) {
      for (let i = 0; i < 10; i++) {
        controller.enqueue(`data: ${i}\n\n`);
        await new Promise(r => setTimeout(r, 1000));
      }
      controller.close();
    }
  });
  return new Response(stream, {
    headers: { "Content-Type": "text/event-stream" }
  });
}
```

```tsx
"use client";
import { useEffect, useState } from "react";

export default function Stream() {
  const [msgs, setMsgs] = useState<string[]>([]);
  useEffect(() => {
    const es = new EventSource("/api/stream");
    es.onmessage = e => setMsgs(m => [...m, e.data]);
    return () => es.close();
  }, []);
  return <ul>{msgs.map((m, i) => <li key={i}>{m}</li>)}</ul>;
}
```

→ AIチャットの**逐次表示**(stream)で必須。

## 14. よくある間違い・エラー(15分)

| 状況 | 間違いの原因 | 直し方 |
|---|---|---|
| CORS エラー | バック側の許可ドメイン未設定 | `allow_origins` にフロントURL追加 |
| `useState` で `"use client"` 忘れ | サーバーコンポーネントで使用 | ファイル先頭に `"use client";` |
| Hydration エラー | サーバーとクライアントで違うHTML | useEffect で初期値を遅延 |
| 環境変数が undefined | NEXT_PUBLIC_ プレフィックスなし | クライアント公開は NEXT_PUBLIC_ |
| Prisma で types エラー | `npx prisma generate` 忘れ | スキーマ変更後は generate 再実行 |
| Tailwind が効かない | `content` 設定漏れ | `tailwind.config.js` の paths 確認 |
| キャッシュで最新データ来ない | デフォルトキャッシュ | `cache: "no-store"` 指定 |
| JWT 検証失敗 | SECRET 不一致 | フロント・バックで共有 |
