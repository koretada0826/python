## 1. 計算量(Big-O)とは(20分)

### 1-1. ざっくり言うと

> **データが増えたとき、どれくらい遅くなるか?を表す記号**

例:お店のレジで、お客さんが10人 → 100人 → 1000人と増えたとき、
- レジが速い店:時間も10倍 → 100倍と**素直に増える**(=O(n))
- レジが遅い店:時間が**100倍 → 10000倍**と爆発する(=O(n²))

| 表記 | 名前 | 例 | 10万件処理する時間の目安 |
|---|---|---|---|
| O(1) | 定数(一瞬) | dict get(辞書から取る) | 0.001秒 |
| O(log n) | 対数(超速い) | 二分探索 | 0.01秒 |
| O(n) | 線形(普通) | for ループ1回 | 0.1秒 |
| O(n log n) | 線形対数(まあまあ) | クイックソート | 1秒 |
| O(n²) | 二乗(遅い) | 二重ループ | 1000秒(17分)|
| O(2^n) | 指数(死ぬ) | 全部分集合 | 宇宙の年齢以上 |

### 1-2. プロの即判断

プロは、コードを見た瞬間に
> 「**このコードO(n²)だから10万件で死ぬ**」
> 「**ここをdict化すればO(1)になって100倍速い**」

と即判断できる。これが面接で問われる。

> **なぜ知ってると単価UP?**
> 経験浅いエンジニアは「動けばOK」と書く → 本番で**サーバー落ちる**。
> Big-Oが分かる人は**最初から事故を防げる**ので、年収に直結する。

### 1-3. よくある間違い

| 間違い | 何がダメ? | 正解 |
|---|---|---|
| `if x in リスト` を1万回ループ | リストの`in`はO(n)で全体O(n²) | setに変えればO(1)で全体O(n) |
| `arr.insert(0, x)` を多用 | リスト先頭挿入はO(n) | dequeのappendleftならO(1) |
| 「動いたからOK」で提出 | 100件では動くが10万件で死ぬ | 計算量を口で説明できるように |

### 2-1. 道具箱の中身

| 道具 | 得意 | 苦手 | 使う場面 |
|---|---|---|---|
| **list** | 末尾追加・番号アクセス | 先頭追加 | ふつうの配列 |

| **heapq** | 最小値取得 | 真ん中の検索 | 優先度つきキュー |
| **dict** | キー検索O(1) | 順番が固定じゃない | キー→値の対応 |

| **Counter** | 個数カウント | - | 出現回数集計 |

### 2-2. list vs deque vs heapq

```python
# list:番号アクセスO(1)、末尾追加O(1)、先頭追加O(n)←遅い
arr = [1, 2, 3]
arr.append(4)        # 末尾追加(速い)
arr.insert(0, 0)     # 先頭挿入(遅い、要素全部ずらす)

# deque:両端アクセスO(1)
from collections import deque
dq = deque([1, 2, 3])
dq.appendleft(0)     # 先頭追加(速い)
dq.popleft()         # 先頭取り出し(速い)

# heapq:最小値の取り出しO(log n)
import heapq
h = []
heapq.heappush(h, 3) # ヒープに入れる
heapq.heappush(h, 1)
heapq.heappush(h, 2)
heapq.heappop(h)     # 最小値の1が出てくる
```

> **イメージ:**
> - **list** = 普通の本棚(番号で取り出すのは速いが、間に挟むのは大変)
> - **deque** = 両端が開いた筒(両側から本を出し入れし放題)
> - **heapq** = いつも一番小さいやつが上にある魔法の山

### 2-3. dict / set(ハッシュテーブル)

```python
# 検索O(1):キーから一瞬で値が出る
d = {"a": 1, "b": 2}
"a" in d  # O(1) ← 一瞬

s = {1, 2, 3}
2 in s    # O(1) ← 一瞬
```

> **超重要なテクニック:**
> 「リストで`in`検索」(O(n))を「setで`in`検索」(O(1))に変えるだけで**100倍速**。
> これだけでLeetCode Easyの3割は解ける。

### 2-4. Counter(個数を数える神道具)

```python
from collections import Counter
c = Counter("aaabbc")
# Counter({'a': 3, 'b': 2, 'c': 1})  ← 自動で個数を数える
c.most_common(2)  # [('a',3),('b',2)] ← 上位2件
```

> **使う場面:**
> - 「文字列の中で一番出てくる文字は?」
> - 「ログの中でアクセス多いユーザーTop10」
> - 「重複チェック」

### 2-5. defaultdict(キーが無くても怒らない辞書)

```python
from collections import defaultdict

# 普通の dict だとキー無しでエラー
d = {}
d["a"].append(1)  # ← KeyError!

# defaultdict なら自動で空リストを作る
d = defaultdict(list)
d["a"].append(1)  # OK!キー無くても勝手にリスト作る
```

> **なぜ便利?**
> グループ分けをするとき、毎回 `if key in d:` を書かなくていい。コードが3行短くなる。

## 3. 主要アルゴリズム(60分)

### 3-1. 二分探索 O(log n)— 「数当てゲーム」の戦略

> **イメージ:**
> 1〜100の数字を当てるゲームで、「真ん中の50」を聞く → 「もっと上」なら75を聞く...
> これだけで**7回以内**に必ず当たる(2^7=128)。

```python
def binary_search(arr, target):
    """ソート済み配列から target を探す。見つかれば位置、無ければ -1"""
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2  # 真ん中の位置を計算
        if arr[mid] == target:
            return mid
        if arr[mid] < target:
            lo = mid + 1  # 真ん中より大きい方を探す
        else:
            hi = mid - 1  # 真ん中より小さい方を探す
    return -1

# 標準ライブラリ版(自分で書かなくてもOKだが、面接では手書きを求められる)
import bisect
bisect.bisect_left(arr, target)
```

> **必須条件:** 配列が**ソート済み**じゃないと使えない。

### 3-2. ソート

```python
arr.sort()                              # 安定ソート O(n log n)
arr.sort(key=lambda x: -x)              # 降順
arr.sort(key=lambda x: (x[1], x[0]))    # 多段ソート(2列目→1列目)
```

> **「安定」って何?**
> 同じ値の要素の順番が、ソート前と後で変わらない、ということ。
> Pythonの`sort()`は安定なので安心。

### 3-3. 再帰(=「関数が自分自身を呼ぶ」)

```python
def fib(n):
    """フィボナッチ数列。1,1,2,3,5,8,13,...のn番目"""
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)  # 自分を呼ぶ

# このままだと O(2^n) で爆遅。fib(40) で数秒
# メモ化で高速化(計算済みの結果を覚えておく)
from functools import cache

@cache
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

# これで O(n) になる。fib(100)も一瞬
```

> **イメージ:**
> 再帰=「鏡の中の鏡」。終了条件(`if n <= 1: return n`)が無いと**無限ループで死ぬ**。

### 3-4. DP(動的計画法)— 「途中の答えを表に書いておく」

> **本質:** 同じ計算を何度もしないで、**1回だけして表に書いておく**。

```python
# 階段問題:n段の階段、1段または2段ずつ登れる。何通りの登り方がある?
def climb(n):
    dp = [0] * (n+1)
    dp[0] = dp[1] = 1
    # i段目の登り方 = (i-1段目の登り方) + (i-2段目の登り方)
    for i in range(2, n+1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

> **なぜ動的計画法と呼ぶ?**
> 「動的に表を埋めていく」から。テーブルに途中結果を書きながら、最終答えに近づいていく。

### 3-5. BFS / DFS(グラフ・木の探索)

```python
# BFS(幅優先探索)— 最短経路を見つけるのに最強
from collections import deque

def bfs(graph, start):
    """startから全頂点を、近い順に訪問"""
    visited = {start}
    q = deque([start])
    while q:
        node = q.popleft()
        print(node)
        for next_node in graph[node]:
            if next_node not in visited:
                visited.add(next_node)
                q.append(next_node)

# DFS(深さ優先探索)— 全パターンを試したい時
def dfs(graph, node, visited=None):
    """startから一本道で深く潜って、突き当たったら戻る"""
    if visited is None:
        visited = set()
    visited.add(node)
    print(node)
    for next_node in graph[node]:
        if next_node not in visited:
            dfs(graph, next_node, visited)
```

> **どっち使う?**
> - **最短経路** → BFS(キュー使う)
> - **全パターン列挙** → DFS(再帰でラク)
> - **迷路で「ゴールまでの最短歩数」** → BFS
> - **「全部のルート列挙」** → DFS

### 3-6. Two Pointers(2本指で挟み撃ち)

```python
# ソート済み配列で、和がtargetになるペアを探す O(n)
def two_sum(arr, target):
    l, r = 0, len(arr) - 1
    while l < r:
        s = arr[l] + arr[r]
        if s == target:
            return (l, r)
        if s < target:
            l += 1   # 和が小さい → 左を右へ(大きくする)
        else:
            r -= 1   # 和が大きい → 右を左へ(小さくする)
```

> **イメージ:**
> 両端から指を1本ずつ出して、目標の数字に向かって挟み撃ち。
> 二重ループ(O(n²))が一重ループ(O(n))になる魔法。

### 3-7. Sliding Window(窓をずらす)

```python
# 連続するK個の最大和を求める
def max_sum_k(arr, k):
    s = sum(arr[:k])  # 最初のK個の合計
    best = s
    for i in range(k, len(arr)):
        # 窓を1つ右にずらす:右の値を足して、左の値を引く
        s += arr[i] - arr[i-k]
        best = max(best, s)
    return best
```

> **イメージ:**
> 列車の窓から景色を見ているような感じ。窓を右にずらすたびに、新しい景色が入って古い景色が消える。

## 4. 木とグラフ(40分)

### 4-1. 二分木(=各ノードが最大2人の子を持つ木)

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left      # 左の子
        self.right = right    # 右の子

# 中順走査(左 → 自分 → 右)— 二分探索木なら昇順で出てくる
def inorder(node):
    if not node:
        return
    inorder(node.left)
    print(node.val)
    inorder(node.right)
```

> **使い道:**
> - 階層構造(組織図、ファイルシステム)
> - 二分探索木(検索O(log n))
> - 式の評価木(計算機の内部)

### 4-3. Dijkstra(最短経路の最強アルゴリズム)

```python
import heapq

def dijkstra(graph, start):
    """start から全頂点への最短距離を計算
    graph[node] = [(隣の頂点, 距離), ...]
    """
    dist = {n: float("inf") for n in graph}
    dist[start] = 0
    pq = [(0, start)]  # (距離, 頂点)のヒープ
    while pq:
        d, node = heapq.heappop(pq)
        if d > dist[node]:
            continue  # 既により短い経路を見つけてる→スキップ
        for next_node, w in graph[node]:
            nd = d + w
            if nd < dist[next_node]:
                dist[next_node] = nd
                heapq.heappush(pq, (nd, next_node))
    return dist
```

> **使い道:** カーナビ、Google Maps、ゲームのAI移動。

## 8. よくある間違い・エラー表

| 症状 | 原因 | 対処 |
|---|---|---|
| `RecursionError: maximum recursion depth` | 再帰の終了条件が無い/深すぎ | `if`で終了条件を入れる、`sys.setrecursionlimit` |
| TLE(Time Limit Exceeded) | 計算量が大きすぎ | O(n²) → O(n)に書き換え、setやdict使う |
| MLE(Memory Limit Exceeded) | 配列を大きく取りすぎ | 不要な変数削除、ジェネレータ使用 |
| `IndexError: list index out of range` | リストの範囲外アクセス | `len()`で確認、境界条件チェック |
| `KeyError` | dictにキーが無い | `.get()`または`defaultdict`使う |
| 答えが微妙にずれる | 整数除算 `/` と `//` を混同 | `//`は切り捨て、`/`は浮動小数 |
