## 2. Clean Architecture(40分)

```
┌──────────────────────────────────────┐
│ Frameworks(FastAPI/SQLAlchemy)       │
│ ┌──────────────────────────────────┐│
│ │ Interface Adapters(Controllers) ││
│ │ ┌────────────────────────────────┐││
│ │ │ Use Cases(create_user)        │││
│ │ │ ┌─────────────────────────────┐│││
│ │ │ │ Entities(User)              ││││
│ │ │ └─────────────────────────────┘│││
│ │ └────────────────────────────────┘││
│ └──────────────────────────────────┘│
└──────────────────────────────────────┘
```

中心ほど**変わらない**、外側ほど**変わりやすい**。
→ DBやフレームワークを差し替えても**コア(ビジネスロジック)は無傷**。

### 実装例

```python
# domain/user.py
@dataclass
class User:
    id: int
    name: str
    email: str

# repositories/user_repository.py(抽象)
class UserRepository(ABC):
    @abstractmethod
    def save(self, u: User): ...
    @abstractmethod
    def find_by_id(self, id: int) -> User: ...

# infrastructure/sql_user_repository.py(具象)
class SQLUserRepository(UserRepository):
    def __init__(self, session): self.session = session
    def save(self, u): ...
    def find_by_id(self, id): ...

# application/create_user.py(ユースケース)
class CreateUser:
    def __init__(self, repo: UserRepository):
        self.repo = repo
    def execute(self, name, email):
        user = User(id=None, name=name, email=email)
        self.repo.save(user)
        return user

# presentation/api.py(FastAPI)
@app.post("/users")
def create(name: str, email: str, db = Depends(get_db)):
    use_case = CreateUser(SQLUserRepository(db))
    return use_case.execute(name, email)
```

→ **テスト時はInMemoryRepositoryに差し替え**できる。

## 3. DDD(ドメイン駆動設計)(40分)

「**業務の専門家と同じ言葉で**コードを書く」

### 用語

| 英語 | 意味 | 例 |
|---|---|---|
| **Entity** | IDで区別される | User, Order |
| **Value Object** | 値で区別される | Money, Address |
| **Aggregate** | 整合性を保つ単位 | Order + OrderItems |
| **Repository** | 永続化抽象 | OrderRepository |
| **Service** | 横断ロジック | PricingService |
| **Domain Event** | 業務イベント | OrderPlaced |

### 例:Money

```python
@dataclass(frozen=True)
class Money:
    amount: Decimal
    currency: str
    
    def __add__(self, other):
        if self.currency != other.currency:
            raise ValueError("通貨が違う")
        return Money(self.amount + other.amount, self.currency)
```

→ `int` で持ってるとバグる(円とドル混同)。**型で守る**。

### 「ユビキタス言語」

業務担当者と話す時の言葉と、コード上の名前を**一致**させる。

❌ `def fn1(x, y):` ← 何やってるか不明
✅ `def calculate_total_with_tax(price, tax_rate):` ← 業務通りの名前

## 4. CQRS(コマンドとクエリ分離)(30分)

「**書き込み**と**読み取り**を別モデルで」

```
[書き込みAPI] → Command → 正規化されたDB
[読み取りAPI] ← Query ← 高速な専用ビュー(Redis/ES)
```

### メリット
- 読み取り専用DBで**爆速**
- 書き込みは整合性厳守、読み取りは速度優先

### 実装例(超簡略)

```python
# Command
class CreateOrderCommand:
    user_id: int
    items: list[OrderItem]

class CreateOrderHandler:
    def handle(self, cmd: CreateOrderCommand):
        # 整合性チェック
        # DBに保存
        # Domain Event 発火
        ...

# Query
class GetOrderHistoryQuery:
    user_id: int

class GetOrderHistoryHandler:
    def handle(self, q: GetOrderHistoryQuery):
        # Read DB (Redis/ES) から高速取得
        ...
```

## 5. Event Sourcing(20分)

「**状態じゃなくイベント列を保存**」

```
通常: users テーブル → name="太郎"  (現在の値だけ)

EventSourcing:
  - UserCreated(name="太郎")
  - UserNameChanged(from="太郎", to="次郎")
  - UserDeactivated()
  → イベント列を再生して現在状態を復元
```

→ **完全な監査ログ・タイムトラベル**。金融・医療で必須。

## 6. SOLID + DI(依存注入)(20分)

```python
# NG: 具体に依存
class OrderService:
    def __init__(self):
        self.db = PostgresDB()  # ← 具体
        self.email = SendGrid()  # ← 具体

# OK: 抽象に依存(DI)
class OrderService:
    def __init__(self, db: Database, email: EmailSender):
        self.db = db
        self.email = email
```
