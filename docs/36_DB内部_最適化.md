## 1. インデックスの仕組み(20分)

### B-tree(基本)

```sql
CREATE INDEX idx_user_email ON users(email);
```

→ 内部で**ソート済み木構造**を作る。検索 O(log n)。

### 効くクエリ

```sql
✅ WHERE email = 'x@y.z'
✅ WHERE email LIKE 'x%'
✅ ORDER BY email

❌ WHERE email LIKE '%@y.z'  ← 後方一致は使えない
❌ WHERE LOWER(email) = 'x@y.z'  ← 関数で加工すると無効
```

### 複合インデックス

```sql
CREATE INDEX idx_users ON users(country, created_at);
```

→ 順番大事。`WHERE country='JP'` は使えるが `WHERE created_at='2026-01-01'` は使えない。

## 2. EXPLAIN で実行計画を見る(20分)

```sql
EXPLAIN ANALYZE
SELECT * FROM users WHERE email = 'x@y.z';
```

| 出力 | 意味 |
|---|---|
| `Index Scan` | インデックス使用 ✅ |
| `Seq Scan` | 全件走査 ❌(遅い) |
| `Bitmap Heap Scan` | 中間 |

→ `Seq Scan` 出てたら**インデックス追加** or **クエリ書き換え**。

## 3. N+1問題(よくある罠)(15分)

```python
# NG: ユーザー1人ごとに別クエリ → 100ユーザーで101クエリ
users = session.query(User).all()
for u in users:
    print(u.orders)  # ← 別クエリ実行!

# OK: 1クエリで JOIN
users = session.query(User).options(joinedload(User.orders)).all()
```

→ **応答時間が10倍速**になる。

## 4. トランザクション・分離レベル(20分)

### ACID

| 文字 | 意味 |
|---|---|
| **A**tomicity | 全部成功 or 全部巻戻し |
| **C**onsistency | 整合性保つ |
| **I**solation | 同時実行で干渉しない |
| **D**urability | コミット後は永続 |

### 分離レベル

| レベル | 同時読書 |
|---|---|
| Read Uncommitted | ダーティリードあり(NG) |
| Read Committed | 普通(PostgreSQL default) |
| Repeatable Read | MySQL default |
| Serializable | 最強・最遅 |

### デッドロック対策

```python
# トランザクションは短く
with session.begin():
    u = session.query(User).with_for_update().filter_by(id=1).first()
    u.balance -= 100
    # commit
```

→ `with_for_update()` で **悲観ロック**。

## 5. 接続プール(15分)

```python
from sqlalchemy import create_engine
engine = create_engine(
    "postgresql://...",
    pool_size=20,        # 通常使用
    max_overflow=10,     # ピーク時追加
    pool_timeout=30,     # 待ちタイムアウト
    pool_pre_ping=True,  # 接続切れ検知
)
```

→ **コネクション枯渇**を防ぐ。SaaSで超大事。

## 6. Read レプリカ(20分)

```
[書込み] → [Master DB]
                ↓ レプリケーション
[読込み] ← [Replica DB] × N台
```

→ **読み取り負荷を分散**。

```python
# 書き込み
master_engine.execute(...)

# 読み取り(ランダムで replica選ぶ)
replica = random.choice(replica_engines)
replica.execute(...)
```
