# Day 38:ネットワーク - TCP/IP・HTTP・DNS・TLS

## 💰 +10-30万(インフラ・セキュリティ)
## ⏱ 15時間

「**接続できません**」の原因を**5分で**特定できる。SES面談頻出。

---

## 小学生説明

ネットは**手紙の郵便局**みたいなもの。
- IPアドレス = 住所
- ポート = 部屋番号
- TCP/IP = 配達ルール
- HTTP = 手紙の中身フォーマット
- DNS = 「○○太郎の家」を住所に変換
- TLS = 手紙を**封筒で密封**(盗み見防止)

---

## 1. OSI 7層モデル(必修)(15分)

| 層 | 役割 | 例 |
|---|---|---|
| 7 アプリ | アプリ通信 | HTTP, SMTP |
| 6 表現 | 暗号化 | TLS |
| 5 セッション | 接続管理 | - |
| 4 トランスポート | データ転送 | TCP, UDP |
| 3 ネットワーク | ルーティング | IP |
| 2 データリンク | 隣接通信 | Ethernet |
| 1 物理 | 信号 | LANケーブル |

→ 「**4層の TCP**」「**3層の IP**」を聞かれたら答えられる。

---

## 2. TCP vs UDP(15分)

| | TCP | UDP |
|---|---|---|
| 信頼性 | ◎(再送) | ✗(投げっぱ) |
| 順序 | ◎ | ✗ |
| 速度 | 普通 | 速い |
| 用途 | HTTP/SMTP | DNS/動画/ゲーム |

## 3. HTTP の中身(20分)

```http
GET /api/users HTTP/1.1
Host: example.com
Authorization: Bearer xxx
Content-Type: application/json

{"name": "太郎"}
```

### ステータスコード(完全版)

```
1xx Informational
2xx Success(200 OK / 201 Created / 204 No Content)
3xx Redirect(301 Moved / 304 Not Modified)
4xx Client Error(400/401/403/404/422/429)
5xx Server Error(500/502/503/504)
```

### HTTP/2, HTTP/3

- HTTP/2:多重化で速い
- HTTP/3:**UDP** ベース(QUIC)、モバイルで強い

→ FastAPI は HTTP/2 対応(uvicorn + http2 オプション)。

## 4. DNS(20分)

```bash
# 名前解決
dig example.com
nslookup example.com

# 各レコード
A      → IPv4 アドレス
AAAA   → IPv6
CNAME  → 別名
MX     → メールサーバー
TXT    → 任意テキスト(SPF/DKIM)
```

### Pythonから

```python
import socket
ip = socket.gethostbyname("example.com")
```

→ **DNS 反映遅延**が原因の障害が多い。`dig` で即確認できる。

## 5. TLS / HTTPS(15分)

### 仕組み

1. クライアント:「TLS 開始」
2. サーバー:**証明書**を送る
3. クライアント:証明書検証(信頼できる認証局のサインか)
4. 共通鍵交換 → 以後暗号通信

### 証明書取得

```bash
# Let's Encrypt(無料)
certbot --nginx -d example.com
```

### Python の HTTPS

```python
import requests
r = requests.get("https://example.com", verify=True)  # 検証あり
```

## 6. ロードバランサ・CDN(15分)

### ALB(AWS)

```
[ALB] ──┬─ [App1]
        ├─ [App2]
        └─ [App3]
```

### CloudFront(CDN)

世界中の**エッジサーバー**にキャッシュ → ユーザーに近い場所から配信。

→ Streamlit Cloud / Render / Vercel は**自動でCDN**してくれる。

## 7. ファイアウォール・セキュリティグループ(15分)

```
[インターネット]
   ↓ (port 80, 443 だけ許可)
[Web サーバー]
   ↓ (port 5432 だけ Web→DB許可)
[DB サーバー]
```

→ **最小権限**(Principle of Least Privilege)。

### AWS Security Group 例

```
Inbound:
  TCP 80    0.0.0.0/0  ← 誰でも HTTP
  TCP 443   0.0.0.0/0  ← 誰でも HTTPS
  TCP 22    XX.XX.XX/32 ← 自分だけ SSH
Outbound:
  ALL  0.0.0.0/0
```

## 8. デバッグツール(15分)

```bash
# 接続確認
curl -v https://example.com
curl -I example.com  # ヘッダーだけ
ping example.com
traceroute example.com

# ポート確認
nc -zv example.com 443

# 詳細パケット
tcpdump -i eth0 port 443
wireshark  # GUI

# サーバー側のリッスン状況
ss -tlnp
netstat -tlnp
```
