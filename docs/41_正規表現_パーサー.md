## 1. 正規表現とは何か(15分)

### 1-1. ふつうの検索 vs 正規表現

普通の `"abc" in text` は「abc という**文字そのもの**」しか探せない。

正規表現は「**こういう特徴を持った文字列**」を探せる。

| やりたいこと | 普通の検索 | 正規表現 |
|---|---|---|
| 「abc」を探す | できる | できる |
| 「数字3桁」を探す | **できない** | `\d{3}` |
| 「メールアドレス」を探す | **できない** | `\w+@\w+\.\w+` |
| 「電話番号(0で始まる)」を探す | **できない** | `0\d{1,4}-\d{1,4}-\d{4}` |

> **比喩:** 普通の検索が「写真と一致する人を探せ」、
> 正規表現が「**身長170cm前後・黒髪・眼鏡の人を探せ**」。
> 後者の方が**柔軟で強力**。

### 1-2. なぜ「正規」表現?
数学の世界で「**規則的に書ける表現**」のこと。難しい話なので、
「**パターンを記号で書く方法**」とだけ覚えればOK。

## 2. 正規表現の基本(20分)

```python
import re  # re は regular expression の略

# search:最初に見つかった1個を返す
re.search(r"\d+", "abc123def")
# → Match: "123"

# findall:全部見つけてリストで返す
re.findall(r"\d+", "1 abc 23 def 456")
# → ['1', '23', '456']

# sub:見つけた箇所を置換
re.sub(r"\d+", "N", "1 abc 23")
# → "N abc N"
```

### `r"..."` の意味(超重要)

```python
"\\d"   # ← バックスラッシュを2回書かないとダメ(面倒)
r"\d"   # ← raw string。そのまま \ を書ける(便利)
```

> **覚え方:** 正規表現は **必ず `r"..."`** で書く。これだけ。

## 3. メタ文字早見表(必修・20分)

| 記号 | 意味 | 例 |
|---|---|---|
| `\d` | 数字(0-9) | `\d{3}` → 3桁の数字 |
| `\w` | 英数字+アンダースコア | `\w+` → 英単語 |
| `\s` | 空白文字 | `\s+` → 連続スペース |
| `.` | 任意1文字(改行以外) | `a.c` → "abc","aXc"... |
| `*` | 直前を**0回以上** | `ab*` → "a","ab","abb"... |
| `+` | 直前を**1回以上** | `ab+` → "ab","abb"...("a"はNG) |
| `?` | 直前を**0回または1回** | `colou?r` → "color","colour" |
| `{n}` | ちょうど n 回 | `\d{4}` → 4桁の数字 |
| `{n,m}` | n〜m 回 | `\d{2,4}` → 2〜4桁 |
| `^` | 行頭 | `^abc` → 行頭の "abc" |
| `$` | 行末 | `\.$` → 行末のピリオド |
| `[abc]` | a/b/c のどれか | `[aeiou]` → 母音 |
| `[^abc]` | a/b/c**以外** | `[^0-9]` → 数字以外 |
| `(...)` | グループ化 | 後で取り出せる |
| `(?P<name>...)` | 名前付きグループ | 名前で取り出せる |

> **覚え方の比喩:**
> - `\d` `\w` `\s` は「**カテゴリの指定**」(数字・文字・空白)
> - `*` `+` `?` `{n}` は「**回数の指定**」
> - `^` `$` は「**位置の指定**」(行頭・行末)
> - `[]` は「**選択肢**」、`()` は「**かたまり**」

## 4. 実務でよく使うパターン(15分)

```python
# 4-1. メールアドレス
EMAIL = r"[\w\.-]+@[\w\.-]+\.\w+"

# 4-2. 日付(2026-04-30 形式)
DATE = r"\d{4}-\d{2}-\d{2}"

# 4-3. 電話番号(0で始まる)
PHONE = r"0\d{1,4}-\d{1,4}-\d{4}"

# 4-4. URL(http または https)
URL = r"https?://[\w\./?=&%-]+"

# 4-5. IPアドレス
IP = r"\b(?:\d{1,3}\.){3}\d{1,3}\b"

# 4-6. 名前付きグループ(取り出しやすい)
m = re.search(r"(?P<year>\d{4})-(?P<month>\d{2})", "2026-04")
m.group("year")   # → "2026"
m.group("month")  # → "04"
```

> **コツ:** 自分で1から書かなくていい。
> クロードに「日本の電話番号にマッチする正規表現を書いて」と頼めば一瞬。
> 大事なのは**読めること**と**動作確認できること**。

## 5. ログ解析の実例(30分)

```python
import re
from collections import Counter

# Apacheアクセスログのパターン
# 例: 192.168.1.1 - - [30/Apr/2026:10:00:00] "GET /api HTTP/1.1" 500 1234
LOG = re.compile(
    r'(\S+) - - \[(.*?)\] "(\w+) (\S+).*?" (\d+) (\d+)'
)
# (\S+)= IP / (.*?)= 日付 / (\w+)= メソッド / (\S+)= URL / (\d+)= ステータス / (\d+)= サイズ

errors = Counter()  # 「URL別の5xxエラー回数」を数える辞書

with open("access.log") as f:
    for line in f:           # 1行ずつ処理(メモリ節約)
        m = LOG.match(line)  # パターン照合
        if m and int(m.group(5)) >= 500:  # ステータスが500以上
            errors[m.group(4)] += 1       # URL でカウントUP

# 5xx エラーが多いURL TOP10
print(errors.most_common(10))
```

→ **GB級のログから5xxエラーTOP10**を1分で出せる。
これが手作業なら丸1日。**だから単価が上がる**。

### よくある間違い(ログ解析)

| 間違い | 起こること | 直し方 |
|---|---|---|
| 全行を `f.read()` で読む | メモリ爆発(GB超) | `for line in f:` で1行ずつ |
| 正規表現が貪欲(`.*`) | 思った以上に長くマッチ | `.*?`(最小マッチ)を使う |
| `\d` の代わりに `.` | 数字以外もマッチして誤検出 | 数字だけなら `\d` |

## 6. HTML/XML パーサー(20分)

### 6-1. 大原則:**HTMLを正規表現でパースしない**

HTML はネスト構造があるので、**正規表現では絶対に正しく解析できない**。
有名な逸話があるレベルでアンチパターン。

→ **必ず BeautifulSoup を使う**。

```python
from bs4 import BeautifulSoup

soup = BeautifulSoup(html, "lxml")  # lxml=速いパーサー

# 全部のリンク
for a in soup.find_all("a"):
    print(a["href"])

# 特定のクラス
soup.find_all("div", class_="price")

# CSS セレクタも使える
soup.select("div.product > h2")
```

### 6-2. なぜ正規表現じゃダメ?

| HTML の例 | 正規表現の弱点 |
|---|---|
| `<a href="x">` のネスト | 入れ子の対応が取れない |
| 属性の順番違い | 順番に依存する正規表現が壊れる |
| コメント `<!-- -->` の中 | 中のタグも誤マッチする |
| 改行・空白の有無 | 全パターンに対応するのが地獄 |

→ **HTML はパーサーに任せる**。これが鉄則。

## 7. 独自形式パーサー(Lark)(40分)

### 7-1. なぜ独自パーサーが必要?

社内独自のログフォーマット、設定ファイル、社内DSL(独自言語)…
こういうものは**正規表現でも辛い**。

そこで **文法を定義** → **自動でパーサー生成**できるのが Lark。

### 7-2. インストール

```bash
pip install lark
```

### 7-3. シンプルな計算式パーサー

```python
from lark import Lark

# 文法を文字列で定義
grammar = """
start: expression
expression: term ("+" term)*
term: NUMBER
%import common.NUMBER
%ignore " "
"""
# start から始まり、expression は term を + でつなげたもの
# term は NUMBER(数字)
# 半角スペースは無視

parser = Lark(grammar)
tree = parser.parse("1 + 2 + 3")
print(tree.pretty())
# 木構造でパース結果を表示
```

→ **DSL**(社内独自言語)を作れる。
config ファイルや、独自クエリ言語などに使える。

### 7-4. どんな案件で使う?

- 独自設定ファイル(`*.conf`)を**安全に読み込む**
- 社内ルールエンジン(条件式を独自記法で書く)
- 業務帳票の**独自テキスト形式**から構造化データへ

## 9. よくあるエラー・落とし穴(10分)

| 症状 | 原因 | 直し方 |
|---|---|---|
| マッチしない | エスケープ忘れ(`.` `(` `+`) | `\.` `\(` `\+` で書く |
| 思ったより長くマッチ | 貪欲マッチ `.*` | `.*?`(最小マッチ)に |
| 改行に困る | `.` は改行にマッチしない | `re.DOTALL` フラグ追加 |
| 大文字小文字 | デフォルトは区別する | `re.IGNORECASE` フラグ |
| 日本語マッチしない | `\w` は日本語非対応な場合あり | `[ぁ-んァ-ン一-龯]+` を使う |
