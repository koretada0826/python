## 1. 統計検定(必修)(30分)

### 1-1. t検定(2群の平均差)

> **使う場面:** 「グループAとB、平均が違う?それとも誤差?」

```python
from scipy import stats

group_a = [100, 110, 95, 105, 98]   # コントロール群
group_b = [120, 115, 130, 125, 118] # 介入群(キャンペーンした方)

# t統計量とp値を計算
t, p = stats.ttest_ind(group_a, group_b)

if p < 0.05:           # p値が5%未満なら
    print("有意差あり")  # まぐれの確率が低い → 本物の効果
else:
    print("有意差なし")  # 偶然でもこれぐらい起こる
```

> **p値の意味:** 「**もし差がないと仮定したとき、これだけの差が偶然出る確率**」。
> 5%未満なら「偶然じゃ説明できないね」と判断する習慣。

### 1-2. カイ二乗検定(クロス集計)

> **使う場面:** 「広告A vs B でCV率に差ある?」(YES/NO データ)

```python
table = [[100, 50],   # 広告A: CV=100, 非CV=50
         [80, 70]]    # 広告B: CV=80, 非CV=70

chi2, p, dof, expected = stats.chi2_contingency(table)

if p < 0.05:
    print("広告で差がある")
```

### 1-3. A/Bテスト(自分で式)

```python
from math import sqrt

def ab_test(visits_a, clicks_a, visits_b, clicks_b):
    """CTR(クリック率)に差があるか判定"""
    ctr_a = clicks_a / visits_a
    ctr_b = clicks_b / visits_b
    pooled = (clicks_a + clicks_b) / (visits_a + visits_b)
    se = sqrt(pooled * (1 - pooled) * (1/visits_a + 1/visits_b))
    z = (ctr_b - ctr_a) / se
    p = 2 * (1 - stats.norm.cdf(abs(z)))
    return p

# 訪問1万人、クリック500 vs 訪問1万人、クリック550
p = ab_test(10000, 500, 10000, 550)
print(p)  # 5%未満なら「Bが本当に良い」
```

→ **本当に効果あったか**を統計で判定。

### 1-4. よくある間違い

| 間違い | 起こること | 直し方 |
|---|---|---|
| サンプルが小さすぎる | 差が出ない/誤判定 | 数百〜数千以上集める |
| p値だけ見る | 「効果量」を無視 | 効果量(Cohen's d)も見る |
| テスト中に何度も覗く | 偽陽性多発 | 期間を決めてから判定 |
| 多重比較放置 | 偶然有意になる | Bonferroni 補正 |

## 2. ベイズ統計(30分)

### 2-1. ベイズって何?

> **比喩:** ふつうの統計 = 「**白黒判定**」(有意 or なし)
> ベイズ = 「**確率で答える**」(Aが勝つ確率は78%)

経営者の質問は「効果あるかどうか」より「**勝つ確率いくつ?**」のほうが直感的。
だからベイズの方が**ビジネスで刺さる**。

### 2-2. PyMC

```bash
pip install pymc
```

```python
import pymc as pm

with pm.Model() as model:
    # 事前分布(何も知らないので一様)
    p_a = pm.Beta("p_a", alpha=1, beta=1)
    p_b = pm.Beta("p_b", alpha=1, beta=1)
    
    # 観測データ
    obs_a = pm.Binomial("obs_a", n=visits_a, p=p_a, observed=clicks_a)
    obs_b = pm.Binomial("obs_b", n=visits_b, p=p_b, observed=clicks_b)
    
    # 差(p_b - p_a)を計算
    diff = pm.Deterministic("diff", p_b - p_a)
    
    # MCMC でサンプリング(2000サンプル)
    trace = pm.sample(2000)

# 「Bが Aより良い確率」が直接出る
prob_b_better = (trace.posterior["diff"] > 0).mean()
print(f"Bが勝つ確率: {prob_b_better*100:.1f}%")
```

→ p値より直感的。**「90%の確率でBが勝つ」**と説明できる → 経営者がすぐ動ける。

## 3. 因果推論(40分)

### 3-1. 相関 ≠ 因果(超重要)

「アイス売上UPと水死者UP」 → 相関ある。
でも**原因**は両方とも**気温**(暑いから泳ぐ&アイス食べる)。

→ アイスの売上を増やしても水死者は増えない。
→ **因果と相関を取り違えると、施策が完全に外れる**。

> **比喩:** 雨の日に長靴履いた人が滑ってない → 「長靴のおかげ」?
> 違う。「**そもそも転びにくい人が長靴を選んでる**」かも。
> こういう**見えない選び方の偏り**を統計で取り除くのが因果推論。

### 3-2. 介入(因果効果を測る方法)

#### A. RCT(ランダム化比較試験)

ユーザーをランダムに2群に分けて、片方だけ介入。
→ **最強だがコスト高**。

#### B. DiD(差分の差分法)

```python
import statsmodels.formula.api as smf

# Before/After × Treatment/Control の4群
# y = 売上, time = 0/1(前後), treatment = 0/1(対象/対照)
model = smf.ols(
    "y ~ time + treatment + time:treatment",
    data=df
).fit()
print(model.summary())
# time:treatment の係数が「純粋なキャンペーン効果」
```

> **DiD のキモ:** 「対照群の自然増加分」を引き算で取り除く。

#### C. 傾向スコアマッチング

```bash
pip install causalml
```

```python
from causalml.inference.meta import LRSRegressor

treatment_effect = LRSRegressor().estimate_ate(X, treatment, y)
# X = 共変量(年齢、性別など), treatment = 0/1, y = 結果
```

#### D. Instrumental Variable(操作変数)

→ ガチの統計学者向け。**「観察データから因果を推定する最終兵器」**。

### 3-3. 因果推論を使う場面

| 場面 | 手法 |
|---|---|
| 新機能をランダムに見せられる | RCT |
| キャンペーン前後の比較 | DiD |
| 既に介入された人と非介入の比較 | 傾向スコア |
| ランダム化できない・できなかった | 操作変数 |

## 4. 時系列予測(40分)

### 4-1. Prophet(Facebook製・最強)

> **イメージ:** 売上・需要などの**未来予測**をいい感じにやってくれる魔法。

```bash
pip install prophet
```

```python
from prophet import Prophet
import pandas as pd

# Prophet は ds(日付)・y(値)の2列が必須
df = pd.DataFrame({"ds": dates, "y": values})

m = Prophet(
    yearly_seasonality=True,   # 年周期(夏に売上↑など)
    weekly_seasonality=True    # 週周期(土日↑など)
)
m.fit(df)

# 30日先までの予測フレーム作成
future = m.make_future_dataframe(periods=30)
forecast = m.predict(future)

m.plot(forecast)  # グラフ化
```

→ **売上予測・需要予測**が1分で。
小売・ECで月10-30万円案件多数。

### 4-2. ARIMA / SARIMA(古典的)

from statsmodels.tsa.arima.model import ARIMA

# (p,d,q): AR次数, 差分, MA次数

## 6. よくあるエラー・落とし穴

| 症状 | 原因 | 直し方 |
|---|---|---|
| p値が極端に小さい | サンプル多すぎ | 効果量(Cohen's d)で見る |
| Prophetが警告 | 日付フォーマット違い | `pd.to_datetime()` で揃える |
| ベイズが収束しない | 事前分布が強すぎ | 一様分布に近づける |
| DiDで結果がブレる | 並行トレンド仮定違反 | 別手法に切り替え |
| MCMCが激遅 | サンプル数多すぎ | チェーン数・サンプル数を減らす |
