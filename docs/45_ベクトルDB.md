## 1. Embedding とは(15分)

文章 → 高次元ベクトル(数字の配列、例: 1536次元)

```python
from openai import OpenAI

resp = client.embeddings.create(
    model="text-embedding-3-large",
    input="Pythonでデータ分析"
)

```

→ 「**意味が近い文章は、ベクトル空間で近い場所**」にある。

## 3. pgvector(超推奨)(30分)

```sql
-- 拡張有効化
CREATE EXTENSION vector;

-- テーブル
CREATE TABLE documents (
    id SERIAL PRIMARY KEY,
    content TEXT,
    embedding vector(1536)
);

-- インデックス(HNSW、超高速)
CREATE INDEX ON documents USING hnsw (embedding vector_cosine_ops);

-- 検索
SELECT content
FROM documents
ORDER BY embedding <=> '[0.1, 0.2, ...]'::vector
LIMIT 5;
```

→ **PostgreSQL 1台**で完結、運用ラク。

## 4. ハイブリッド検索(20分)

```python
# ベクトル検索 + キーワード検索を統合
def hybrid_search(query, k=10):
    # 1. ベクトル
    vec = embed(query)
    vector_results = db.execute("""
        SELECT id, content, 1 - (embedding <=> %s::vector) as score
        FROM documents ORDER BY embedding <=> %s::vector LIMIT %s
    """, (vec, vec, k))
    
    # 2. BM25(全文検索)
    bm25_results = db.execute("""
        SELECT id, content, ts_rank(to_tsvector(content), to_tsquery(%s)) as score
        FROM documents WHERE to_tsvector(content) @@ to_tsquery(%s)
        ORDER BY score DESC LIMIT %s
    """, (query, query, k))
    
    # 3. RRF で統合
    return rrf_combine(vector_results, bm25_results)
```

→ 精度が**飛躍的に**上がる。

## 5. Re-ranking(15分)

```python
from sentence_transformers import CrossEncoder
reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-12-v2")

# Top 100 候補 → Re-rank で Top 10 に絞る
candidates = vector_search(query, k=100)
pairs = [(query, doc.content) for doc in candidates]
scores = reranker.predict(pairs)
top10 = sorted(zip(candidates, scores), key=lambda x: -x[1])[:10]
```

→ **2段階検索**で精度UP。

## 6. Chunking 戦略(15分)

文書を細切れにする戦略が**精度を決める**。

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\n\n", "\n", "。", " ", ""]
)
chunks = splitter.split_text(text)
```

→ **段落・文単位**で切る、**重なり**で文脈保持。
