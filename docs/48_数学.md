## 1. 線形代数(必修)(40分)

### 1-1. ベクトル

> **比喩:** ベクトル = 「**矢印**」or 「**数字の並び**」。

```python
import numpy as np

# ベクトル(1次元)
v = np.array([1, 2, 3])

# 大きさ(ノルム)
np.linalg.norm(v)   # √(1²+2²+3²)

# 内積(2つのベクトルの「向きの近さ」を表す)
np.dot(v, v)        # 1*1 + 2*2 + 3*3 = 14
```

### 1-2. 行列

> **比喩:** 行列 = 「**ベクトルを縦に積んだもの**」、または「**変換装置**」。

```python
# 2x2 行列
A = np.array([[1, 2],
              [3, 4]])

# 行列とベクトルの掛け算(変換)
v = np.array([1, 1])
A @ v               # [1*1+2*1, 3*1+4*1] = [3, 7]

# 転置(縦横入れ替え)
A.T

# 逆行列
np.linalg.inv(A)

# 固有値・固有ベクトル(行列の本質を捉える)
eigenvalues, eigenvectors = np.linalg.eig(A)
```

### 1-3. ML で「行列・ベクトル」が登場する場面

| 場面 | 数学 |
|---|---|
| Embedding 類似度 | **ベクトル内積** |
| ニューラルネット | **行列×ベクトル** + 活性化 |
| CNN(畳み込み) | **テンソル積** |
| 主成分分析(PCA) | **固有値分解** |
| 推薦エンジン | **行列分解(SVD)** |

→ 「**ML = 行列演算の積み重ね**」と思ってOK。

## 2. 微分・最適化(30分)

### 2-1. 微分とは

> **比喩:** 微分 = 「**変化の速さ**」を測る。
> 坂道の傾き、車のスピード、株価の変動率…全部微分。

### 2-2. 勾配降下法(ML の学習の心臓)

```python
# y = (w-3)² を最小化したい(答えは w=3)
def loss(w):
    return (w - 3)**2

w = 0          # スタート地点
lr = 0.1       # 学習率(1歩の大きさ)

for _ in range(100):
    grad = 2 * (w - 3)   # 微分(=傾き)
    w = w - lr * grad    # 傾きの逆方向に1歩進む

print(w)  # 約 3.0(最小値に到達)
```

> **図解で理解:**
> グラフが谷(U字)。今いる位置の傾きを見て**逆向きに歩く**。
> これを繰り返すと**谷の底**(最小値)に着く。

### 2-3. ML の本質

> **ML の学習** = 「**損失関数を最小化する勾配降下法**」だけ。
> あとは派生(SGD, Adam, RMSProp など)。

### 2-4. 自動微分(autograd)

```python
import torch

x = torch.tensor(2.0, requires_grad=True)  # 微分対象
y = x**2 + 3*x + 1                          # 関数

y.backward()                                # 自動で微分
print(x.grad)                               # 7.0(=2x+3 を x=2 に代入)
```

→ PyTorch は**式を自動で微分**してくれる。
人間は**式を書くだけ**でOK。

## 3. 確率(20分)

### 3-1. 基本

```python
from scipy import stats

# 正規分布
stats.norm.pdf(0, loc=0, scale=1)  # 平均0・標準偏差1の点 0 における確率密度
stats.norm.cdf(1.96)               # 累積分布(=1.96以下の確率)

# 1000個サンプル生成
samples = stats.norm.rvs(size=1000)
```

### 3-2. ベイズの定理

```
P(A|B) = P(B|A) × P(A) / P(B)
```

> **比喩:** 「メールに『100万円当選』とあったとき、それが迷惑メールである確率は?」
> 普通の感覚と数式が一致するように設計されてる。

ナイーブベイズで迷惑メール判定がこの定理の応用。

### 3-3. 主な分布

| 分布 | 用途 |
|---|---|
| 正規分布(ガウス) | 自然現象・誤差 |
| ベルヌーイ分布 | YES/NO(コイン) |
| 二項分布 | n回中k回成功 |
| ポアソン分布 | 単位時間あたり発生回数 |
| ベータ分布 | 確率の確率(ベイズで使う) |

## 4. 情報理論(15分)

### 4-1. エントロピー

「**不確実さの量**」を数値化する指標。

```python
import numpy as np

def entropy(p):
    return -np.sum(p * np.log2(p + 1e-10))

# コインが完全にランダム(50/50)→ エントロピー最大
entropy(np.array([0.5, 0.5]))   # 1.0
# 完全に偏ってる(100/0)→ エントロピー0
entropy(np.array([1.0, 0.0]))   # 0.0
```

### 4-2. 交差エントロピー(分類のloss)

```python
def cross_entropy(y_true, y_pred):
    return -np.sum(y_true * np.log(y_pred + 1e-10))
```

> **意味:** 「予測が正解にどれだけ近いか」を測る。
> 分類タスクのlossとして必ず登場。

→ ML の分類 = **交差エントロピーを最小化する**作業、と覚える。
