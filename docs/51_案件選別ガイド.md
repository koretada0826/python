# Day 51:案件選別ガイド - 即金最速の選び方

## 💡 これは何?(小学生でもわかる説明)

**「応募してはいけない案件」の見分け方**を覚える日。

### なぜ大事?
- 全部の案件に応募してたら**時間がいくらあっても足りない**
- ハズレ案件を引くと「**時給300円**」「**お金もらえない**」「**★1つけられる**」
- ハズレを避けるだけで **効率が3倍** になる
- 同じ努力で**収入が変わる**のは、案件を選ぶ目があるかどうか

### イメージ
- 案件選び=**釣り**
- 海には **マグロ(高単価)** と **雑魚(低単価)** と **フグ(毒=詐欺)** が泳いでる
- 全部釣ろうとすると、雑魚で時間が消えて、フグで死ぬ
- **マグロだけ狙う**=賢い釣り人

### Claude Code に何を頼めばいいか
- 「この案件説明、危ない部分ない?チェックして」
- 「この依頼に質問するなら何を聞くべき?」
- 「クライアントへの返信文を書いて(丁寧モード/堅めモード)」

## 1. 信号別判定(=「案件の見た目で3秒判断」)(20分)

> **イメージ:** 道路の信号と同じ。青なら進む、赤なら止まる。
> 案件にも「青信号」「黄色信号」「赤信号」がある。

### 🟢 GO(積極応募)=青信号

これが揃ってる案件だけ応募してOK。

- **仕様が明確**(input/output/件数が書いてある)
- **予算 5,000円以上**(初回は1万円以上が安心)
- **クライアント評価 ★4以上**(または評価なしの新規でもOK)
- **本人確認済み**(運営が身元チェック済)
- **締切が1週間以上先**(急ぎすぎない)
- **応募数 5〜20件**(0は不安、50超は競争激しすぎ)

> **なぜ?**
> 仕様明確=後から「思ってたのと違う」が起きにくい。
> 締切1週間以上=焦らずに作れる。
> 応募数5〜20=ライバルが少なくて受注しやすい黄金ゾーン。

### 🟡 注意(見極め必要)=黄色信号

ちゃんと質問してから判断。

- 仕様あいまい → **質問して返事が早ければ応募**
- 予算 5,000円未満 → **量で稼げる(1時間で終わる)なら応募**
- 評価なし → **メッセージのやりとりで雰囲気を確認**

> **コツ:** 質問して**返信が遅い人・要点ずれてる人**は黄色→赤に変わる。
> その時点で降りる。

### 🔴 NG(絶対NG)=赤信号

これらは **どんなに金額が良くても応募しない**。

- 「**直接連絡先教えて**」(LINE/メール) → 規約違反 + 詐欺率99%
- 「**テスト納品してから判断**」 → タダ働き踏み倒し
- **完全成果報酬**(売れたら払う) → 工数かけて0円リスク
- **1記事500円**みたいな量産案件 → 時給500円で消耗
- 「**コピペで月10万!**」 → 100%スパム
- 「**月100時間で月3万**」 → 時給300円(コンビニバイト以下)
- **日本語が怪しい依頼者** → 海外詐欺グループの可能性

> **なぜ「直接連絡先」が危ない?**
> ランサーズ/クラウドワークス経由なら**運営が間に入って守ってくれる**。
> LINE直接 = **証拠がない・支払い保証もない**。
> トラブった時に「**そんな話してない**」と言われて終わる。

## 2. 単価判断フロー(=「時給で考える」)(15分)

### 2-1. たった1つの公式

```
時給換算 = 提示金額 ÷ 想定工数(時間)
```

これだけ覚えれば全案件を比較できる。

| 時給 | 判定 |
|---|---|
| **5,000円以上** | 🟢 GO(理想) |
| 3,000〜5,000円 | 🟡 評価★0の最初だけGO |
| 1,000〜3,000円 | 🟠 評価作りの最初3件のみ |
| 1,000円未満 | 🔴 絶対NG |

### 2-2. 具体例で計算してみる

**例1:** 「Excel統合スクリプト 5,000円」

```
クロードに頼めば1時間で完成
→ 5,000円 ÷ 1時間 = 時給5,000円
→ 🟢 GO!
```

**例2:** 「Webスクレイピング 月10サイト 月3,000円」

```
スクレイピングは1サイト5時間 × 10 = 月50時間
→ 3,000円 ÷ 50時間 = 時給60円
→ 🔴 絶対NG(コンビニの100分の1)
```

**例3:** 「PDFからデータ抽出 30,000円」

```
PDFパース系は3〜4時間で済む
→ 30,000円 ÷ 4時間 = 時給7,500円
→ 🟢 GO!(高単価)
```

> **なぜ時給で考える?**
> 金額だけ見ると「**5万円!**」に飛びつくけど、
> 100時間かかる案件なら時給500円。
> **「金額が大きい = 良い案件」ではない**。

## 3. クライアントの見極め方(=「人柄チェック」)(15分)

### 3-1. 質問してみる(これが超重要)

応募前 or 応募直後に**必ず質問する**。

```
ご依頼ありがとうございます。
納品物を明確にするため、以下を教えてください:

1. データ件数の想定(10件 / 100件 / 1000件)
2. 出力形式(Excel / CSV / JSON)
3. 納期の優先度(早さ重視 / 品質重視)

お忙しい中恐れ入りますが、よろしくお願いします。
```

### 3-2. 返信で見抜く「良い客 vs 悪い客」

| 返信パターン | 判定 |
|---|---|
| 24時間以内に箇条書きで明確 | 🟢 良い客 |
| 1〜2日かかるが内容は丁寧 | 🟡 普通 |
| 「適当でいいです」 | 🟠 トラブル予兆 |
| 1週間以上返信なし | 🔴 進行で揉める |
| 質問の答えになってない | 🔴 仕様変更の嵐になる |

> **降りる勇気を持つ。**
> 違和感あるクライアントは、**作業始めてから10倍ストレス溜まる**。
> 受注前にやめるのが一番ダメージ少ない。

## 4. 受注しやすい案件の特徴(=「狙い目リスト」)(15分)

### 即金案件 BEST 5

| 案件タイプ | 単価 | 完成時間 | 難易度 |
|---|---|---|---|
| **複数Excel統合** | 8,000〜15,000円 | 1〜2時間 | ★ |
| **PDFからデータ抽出** | 10,000〜30,000円 | 2〜4時間 | ★★ |
| **特定サイトのスクレイピング** | 15,000〜30,000円 | 3〜5時間 | ★★ |
| **CSV→グラフレポート** | 20,000〜50,000円 | 半日 | ★★ |
| **AI要約・翻訳ツール** | 30,000〜100,000円 | 1日 | ★★★ |

→ **これらに集中**すれば月20〜50万。

> **なぜこの5つ?**
> - クロードが一瞬で書ける
> - クライアントの困りごとが**お金になる重さ**
> - 似た案件が**何度も発注される**(リピート受注しやすい)

## 5. 危険な案件の見抜き方(=「詐欺パターン図鑑」)(15分)

### 5-1. 詐欺パターン

**パターンA:情報商材への誘導**

```
タイトル:「コピペするだけで月10万」
本文  :「LINE @xxx で詳細送ります」

→ 100% 詐欺。連絡先請求は規約違反。
```

**パターンB:自社情報商材の販売員**

```
タイトル:「Python講師募集 月20万」
本文  :「教材は当社が用意。SNSで集客するだけ」

→ あなたが詐欺の片棒担がされる側。要注意。
```

**パターンC:海外詐欺グループ**

```
タイトル:「pythonのプログラマーを探していました」
本文  :日本語の助詞がおかしい / 機械翻訳臭

→ 完成しても支払われない。NG。
```

### 5-2. 踏み倒しパターン

```
「完成したら払います」
「テスト納品で見せてください」
「サンプル先に作ってもらえますか」

→ 仮払いない案件は受けない。
   ランサーズ/クラウドワークスの「仮払い」は必須。
```

> **仮払いって何?**
> クライアントが**先にお金を運営に預ける仕組み**。
> 預けてないクライアント=払う気ない可能性。
> 仮払い済まで作業を始めない=鉄則。

## 6. 受注後の品質チェック(=「納品前の最終確認」)(10分)

納品ボタン押す前に**必ず**:

- ☐ サンプル動作確認(**3パターン以上**でテスト)
- ☐ 不正データでクラッシュしない(空・null・異常値)
- ☐ README に**使い方を書く**(セットアップ・実行コマンド・例)
- ☐ requirements.txt 最新化(`pip freeze > requirements.txt`)
- ☐ 余計なファイル削除(`.DS_Store`, `__pycache__`, `.env`)
- ☐ コメント整理(意味不明な作業メモは削除)
- ☐ 日本語動作確認(`encoding="utf-8"` で文字化け対策)

> **なぜここまでやる?**
> 納品後に「動かない」と言われると **5倍時間がかかる**。
> しかも**評価★が下がる**。
> 5分の確認で1時間の修正対応を防げる。

## 7. クレーム対処(=「揉めた時の冷静術」)(20分)

### 7-1. よくあるクレーム3種

#### クレーム① 「動かないんですけど」

```
動作確認の手順を教えてください:
1. Pythonバージョン (python3 --version の結果)
2. 実行コマンドと操作手順
3. エラーメッセージのスクリーンショット

これがあれば即解決します。
```

> **コツ:** **冷静に質問**で時間稼ぎつつ、原因を特定。
> 9割は「環境の違い」か「使い方の違い」で、コードのバグではない。

#### クレーム② 「想定と違う」

```
ご依頼当時の仕様書を確認しました。
〇〇の動作は当初仕様書の3行目に基づいています。

仕様変更でしたら、追加見積もりさせていただきます。
追加分の作業内容と納期をご相談させてください。
```

> **当初契約と違う要求**は別案件として再見積。
> サービス対応で受けるとキリがない。**契約書類が証拠**。

#### クレーム③ 「★1付けます」と脅される

```
ランサーズ運営に通報。
契約完了して支払い受けてから対応。
```

→ 規約違反の脅迫は**運営に依頼**。自分で対応しなくていい。

### 7-2. 揉めた時の鉄則

| やる | やらない |
|---|---|
| 必ず**メッセージ機能**で記録残す | LINE/電話で済ませる |
| **事実だけ**冷静に書く | 感情的に反論する |
| **運営に相談**する | 1人で抱え込む |
| 仮払い済みを確認 | 先に作業し直す |

> **なぜチェックリスト?**
> 人間は「金額」「タイトル」だけ見て**衝動的に応募**してしまう。
> リストでクールダウン=冷静になれる。

## 9. よくある間違い表

| 間違い | 何が起きる | 直し方 |
|---|---|---|
| 全部の案件に応募 | 時間消える・低単価で消耗 | 信号で選別 |
| 金額だけ見て応募 | 工数100時間で時給500円 | **時給で計算** |
| 仕様あいまいでも受注 | 「思ってたのと違う」で揉める | 質問で詰める |
| 仮払い前に作業開始 | 支払われない | 仮払い完了まで待つ |
| クレームに即反論 | 感情的になり★1食らう | **冷静に質問**で返す |
| LINE 直接連絡に応じる | 規約違反&詐欺リスク | 運営内メッセージ徹底 |

## 💼 この章まで終わったら受けられる案件

### 受けられる案件(具体例)
- **悪い案件を回避することで実質時給UP**:同じ稼働時間で月収+20〜50%
- **良案件への集中応募 → ★5評価が貯まりやすくなる**:単価交渉力UP
- **継続案件・月額契約に絞る運用**:単発案件と比べ営業コスト大幅減

### クロードへの頼み方(営業文を書かせる例)
```
「危険な案件・低単価案件を断る丁寧な辞退メール」のテンプレを書いて。
状況:Day 51 で習得した案件選別ガイドに照らして、提示単価が時給1,500円を切るので断る
内容:
・感謝を伝える
・自分の現在の最低時給ライン(または最低単価)を明示
・もし条件改善するなら再交渉可能と書く
・他のクライアントを紹介できる場合は紹介の道を残す
トーン:角を立てない、プロフェッショナル
300字
```
