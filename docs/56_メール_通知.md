# Day 56:メール・通知配信 - SaaS / Webhook

## 💡 これは何?(小学生でもわかる説明)

**ユーザーにメール・通知・LINE で連絡を送る**仕組みを作る日。

### なぜ大事?
- SaaS で**ユーザーへの連絡なし=サービスが死ぬ**
- 登録時の「ようこそメール」、決済時の「領収書」、解約時の「引き止めメール」
- 通知があるだけで**継続率が30〜50%上がる**
- 連絡できない=ユーザーは存在を忘れて去っていく

### イメージ
- メール配信 = **手紙の配達員**
  - Resend = 最新の配達員(早い・安い)
- Webhook 受信 = **荷物の受付係**
  - Stripe や GitHub からの通知を受けて、自動で動く
- Push 通知 = **スマホを揺らす肩たたき**
- LINE 通知 = **若者の心に直接届く郵便**

> **もう1つの例え:**
> メール=**お便り箱に入る紙の通知**
> Push=**スマホ画面のチカチカ**
> LINE=**友達からのトーク**(開封率が一番高い)

### Claude Code に何を頼めばいいか
- 「Resend で会員登録メールを送る Python コードを書いて」
- 「Jinja2 テンプレートで決済完了メールのHTMLを作って」
- 「Stripe Webhook を受信して内部処理を呼ぶエンドポイント書いて」
- 「LINE Messaging API でユーザーに通知送るコード書いて」

## 1. Resend(=「最新の手紙配達員」最推奨)(20分)

### 1-1. なぜ Resend?

- **無料プラン月3,000通**(個人SaaSなら十分)
- **API がシンプル**(SendGrid より楽)
- **配信率が高い**(迷惑メール判定されにくい)
- **HTMLメールも普通の文字メールも対応**

### 1-2. インストール

```bash
pip install resend
```

### 1-3. 最小コード

```python
import resend
import os

# 認証情報
resend.api_key = os.getenv("RESEND_API_KEY")

# 送信
resend.Emails.send({
    "from": "noreply@example.com",          # 送信元(ドメイン認証必要)
    "to": ["user@example.com"],              # 宛先(複数可)
    "subject": "ようこそ",                   # 件名
    "html": "<h1>登録完了</h1><p>サービスへようこそ</p>",  # 本文HTML
})
```

> **準備:** 
> Resend ダッシュボードで**送信ドメイン認証**(SPF/DKIM レコード追加)が必要。
> 認証なしだと送れる相手が制限される。

## 2. テンプレート(Jinja2)(=「メールの差し替え式雛形」)(15分)

### 2-1. なぜテンプレ?

> 「{{name}}さん、ようこそ」みたいに**ユーザー毎に内容を変えたい**。
> 文字列結合(`f"..."`)でやると複雑になる→Jinja2 を使う。

### 2-2. コード

```python
from jinja2 import Environment, FileSystemLoader

# emails/ フォルダから雛形を探す設定
env = Environment(loader=FileSystemLoader("emails"))

# welcome.html.j2 を読み込み
template = env.get_template("welcome.html.j2")

# 変数を流し込んでHTML生成
html = template.render(name="太郎", url="https://app.com/start")

# 送信
resend.Emails.send({
    "from": "noreply@example.com",
    "to": ["taro@example.com"],
    "subject": "ようこそ",
    "html": html,
})
```

`emails/welcome.html.j2`:
```html
<h1>{{ name }}さん、ようこそ</h1>
<p>下のリンクからサービスを開始してください:</p>
<p><a href="{{ url }}">サービスを開始する</a></p>
```

### 2-3. 揃えるテンプレ集

| ファイル名 | 用途 |
|---|---|
| `welcome.html.j2` | 会員登録歓迎 |
| `verify.html.j2` | メール認証 |
| `payment_success.html.j2` | 決済完了/領収書 |
| `payment_failed.html.j2` | 決済失敗 |
| `cancel.html.j2` | 解約完了 |
| `reset_password.html.j2` | パスワードリセット |
| `notify.html.j2` | お知らせ |
| `goodbye.html.j2` | 退会 |

## 3. Webhook 受信(=「荷物の受付係」)(30分)

### 3-1. なぜ?

> Stripe / GitHub / Slack 等は**「○○が起きたよ」とあなたのサーバーに通知**してくる。
> その通知を受け取って自動処理。

### 3-2. Stripe Webhook(再掲・整理)

```python
from fastapi import Request, HTTPException
import stripe

@app.post("/webhook/stripe")
async def stripe_webhook(request: Request):
    payload = await request.body()
    sig = request.headers["stripe-signature"]
    
    # 本物のStripe からの通知か検証
    try:
        event = stripe.Webhook.construct_event(payload, sig, SECRET)
    except stripe.error.SignatureVerificationError:
        raise HTTPException(400)
    
    # イベント別処理
    if event["type"] == "checkout.session.completed":
        # ユーザーをプレミアム化
        ...
    elif event["type"] == "invoice.payment_failed":
        # 失敗メール送信
        ...
    
    return {"ok": True}
```

### 3-3. 共通の鉄則

| やる | やらない |
|---|---|
| **署名検証**を必ず | 検証スキップ |
| `event["id"]` で**重複防止** | 同じイベントで二重処理 |
| 処理は**5秒以内**に返す | 重い処理を同期実行 |
| 重い処理は**キュー(Celery等)に投げる** | エンドポイント内で完結 |

> **なぜ5秒?**
> Stripe / GitHub は**5〜10秒返答ないと再送**してくる。
> 重い処理は別プロセスに任せる。

## 4. Push 通知(=「スマホの肩たたき」)(15分)

### 4-1. ブラウザ通知(Web Push)

```typescript
// 許可を求める
Notification.requestPermission().then(perm => {
  if (perm === "granted") {
    new Notification("新着メッセージ", {
      body: "○○さんからメッセージが届きました",
      icon: "/icon.png",
    });
  }
});
```

> **使いどころ:** チャット・SaaS のお知らせ・SNS。

### 4-2. モバイル通知(FCM = Firebase Cloud Messaging)

```python
from firebase_admin import messaging, credentials, initialize_app

# 初期化(秘密鍵JSONを使用)
cred = credentials.Certificate("firebase-key.json")
initialize_app(cred)

# 通知メッセージを作成
message = messaging.Message(
    notification=messaging.Notification(
        title="新着メッセージ",
        body="○○さんから...",
    ),
    token=user.fcm_token,  # クライアントから取得して保存しておく
)

# 送信
messaging.send(message)
```

> **iOS/Android アプリ**に通知を送る標準。**無料**。

## 5. SMS(Twilio)(=「電話番号宛のテキスト」)(15分)

### 使いどころ

- **二要素認証(MFA)** で「認証コード:123456」を送る
- 緊急通知(深夜の異常アラート)

```bash
pip install twilio
```

```python
from twilio.rest import Client
import os

# 認証
client = Client(os.getenv("TWILIO_SID"), os.getenv("TWILIO_TOKEN"))

# 送信
client.messages.create(
    to="+819012345678",        # 宛先(国番号必須)
    from_="+1...",             # Twilio で取得した送信元番号
    body="認証コード: 123456",
)
```

> **コスト注意:** SMS は**1通5〜10円**。メール(無料〜)より高い。
> 二要素認証など**重要な場面だけ**に。

## 6. LINE Messaging API(=「友達トーク式の通知」)(15分)

### 6-1. なぜ最強?

- **開封率70%超**(メールは20%程度)
- 日本のB2C SaaS で**圧倒的に強い**
- LINE 公式アカウント経由で送信

### 6-2. インストール

```bash
pip install line-bot-sdk
```

### 6-3. コード

```python
from linebot.v3.messaging import (
    Configuration, ApiClient, MessagingApi,
    PushMessageRequest, TextMessage,
)
import os

# アクセストークン設定
config = Configuration(access_token=os.getenv("LINE_TOKEN"))

with ApiClient(config) as api_client:
    api = MessagingApi(api_client)
    api.push_message(PushMessageRequest(
        to="USER_ID",                              # LINE のユーザーID
        messages=[TextMessage(text="お知らせです")],
    ))
```

> **`USER_ID` の取得:**
> ユーザーが公式アカウントを友達追加した時に**Webhook で送られてくる**。
> DB に保存しておく。

→ B2C SaaS で**プッシュ率70%**(メールより高い)。

## 7. よくあるエラーと対処

| 症状 | 原因 | 直し方 |
|---|---|---|
| Resend 401 Unauthorized | API key 間違い | `.env` 確認 |
| Resend 422 Unprocessable | from の認証ドメイン未設定 | DNS にSPF/DKIM追加 |
| Jinja2 `TemplateNotFound` | パス・拡張子間違い | `FileSystemLoader` の指定見直し |
| Webhook 署名検証失敗 | シークレットの不一致 | 環境変数を再確認 |
| Push 通知出ない | 許可されてない | Notification.permission を確認 |
| FCM `Invalid registration token` | トークン古い | 端末で再取得 |
| Twilio エラー | 番号が国番号なし | `+81` 等を頭に |
| LINE `Invalid user ID` | 友達追加されてない | 友達追加 → ID を再保存 |
| メール届かない(迷惑メール扱い) | SPF/DKIM 未設定 | DNS 設定+独自ドメイン使用 |
