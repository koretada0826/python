### なぜ大事?

- 1度作れば**寝てる間にも仕事が来る**仕組みが回る

### イメージ
- 普通のエンジニア = **どこにでもあるラーメン屋**
  - 美味しいけど、目立たない=単価400円
- ブランドのあるエンジニア = **行列の有名店**
  - 美味しさは同じでも、**「あの店主に作ってほしい」**=単価2,000円
  - お客様が**並んででも食べたい**状態

> **もう1つの例え:** 
> 普通=スーパーで売ってる果物
> ブランド=**「○○農園のりんご」**=同じりんごでも3倍の値段で売れる

## 1. ポジショニング(=「何屋さんか1行で言う」)(20分)

### 1-1. 重要性

**「何の人」と思われたいか**を1行で決める。
これが曖昧だと「便利屋」=安く使われる。

### 1-2. 比較

```
❌ 「Pythonエンジニアです」 
   → 100万人いる/差別化ゼロ/単価決まらない

✅ 「EC事業者向けに業務自動化やってます、月20時間削減できます」
   → 唯一無二/具体的/数字で説得力
```

> **なぜ具体例があるとブランド化する?**
> 「Pythonできます」=競合100万人=値下げ競争
> 「EC向けに月20時間削減」=競合10人=**お客様の方から探しに来る**

### 1-3. ポジショニング公式

```
[業界] × [専門技術] × [成果]

例:
EC × Python自動化 × 月20時間削減
不動産 × LLMチャットボット × 問い合わせ50%自動応答
医療 × FastAPI ダッシュボード × カルテ集計自動化
士業 × OCR + AI要約 × 書類処理10倍速
SaaS × Stripe決済設計 × 課金トラブル0件
```

### 1-4. 自分用に書いてみる

| 軸 | 候補 |
|---|---|
| 業界 | EC / 不動産 / 医療 / 士業 / SaaS / 教育 / 製造 / 飲食 |
| 技術 | AI / 自動化 / 分析 / システム / RPA / データ可視化 |
| 成果 | 削減時間 / 売上UP / コスト削減 / トラブル削減 |

→ **3つを掛け算**して**自分の肩書を1行**にする。

## 2. SNS 戦略(深掘り)(=「Twitter/X 攻略法」)(40分)

### 2-1. 投稿の5つの型

| 型 | 例 | 反応 |
|---|---|---|
| **数字付き成果** | 「クライアント年間60万削減した自動化」 | ★★★★★ |
| **失敗談** | 「初案件で◯◯ミスった話」 | ★★★★ |
| **具体tips** | 「pandas で複数Excel統合する3行コード」 | ★★★★ |
| **質問・問いかけ** | 「みんなどのDBクライアント使ってる?」 | ★★★ |
| **共感** | 「フリーランスあるある」 | ★★ |

> **比率の目安:**
> 成果4:Tips3:失敗談2:問いかけ1:共感1 のバランス。
> 全部が「成果報告」だと自慢ぽくて嫌われる。

### 2-2. バズる投稿の構造

```
[1行目] 結論・驚きで掴む
       ↓
[2-5行目] 詳細・数字でリアル感
       ↓
[最終行] CTA(コメント誘導・リンク・続編予告)
```

### 2-3. 実例

```
クライアントの月20時間の作業を Python で自動化したら、
1分で終わるようになった。

人件費換算で月60,000円削減。
→ 年間72万円のコストカット。

開発期間: 5日
工数  : 計15時間
コード : 約400行

詳細をブログに書いた → [URL]
```

> **なぜこれがバズる?**
> 1. **具体的な数字**(60,000円・15時間・400行)
> 2. **時間と金額**両方=説得力
> 3. **続きが見たくなるリンク**=エンゲージメント

### 2-4. プロフィール最適化

```
名前 / @ハンドル / アイコン / プロフィール文 / ヘッダー画像 / 固定ツイート
```

| 項目 | コツ |
|---|---|
| 名前 | 本名 or 専門領域がわかる名前 |
| アイコン | **顔写真 or プロが描いたイラスト**(LINEアバターはNG) |
| プロフィール文 | **ポジショニング1行 + 実績数字 + 連絡先** |
| ヘッダー | サービス紹介 or 実績スクショ |
| 固定ツイート | **一番反応が良かった投稿** or 自己紹介スレッド |

## 3. note ブログ戦略(=「ストック型コンテンツ」)(30分)

### 3-1. なぜ note?

- **SEOで検索流入**が継続(Twitterは流れて消える)
- **長文で深い情報**=信用構築
- 1記事が**1年後にも案件を呼ぶ**

### 3-2. 月10記事ルール

最初の3〜6ヶ月は**月10記事**=量で押す。

| カテゴリ | 例 |
|---|---|
| **SEO狙い** | 「Python pandas Excel統合 自動化」 |
| **ストーリー** | 「初案件で50万損した話」 |
| **ノウハウ** | 「AI Bot 構築 5ステップ」 |
| **比較レビュー** | 「LangChain vs LlamaIndex」 |
| **チュートリアル** | 「FastAPI + Stripe 課金システム実装」 |

### 3-3. 記事の型

```
1. タイトル(検索キーワード入り)
2. 導入  : 誰のどんな悩みを解決
3. 結論  : この記事で何が分かる(先出し)
4. 本論  : 5〜7セクション(コード付き)
5. まとめ + CTA(問い合わせ・Twitterフォロー)
```

> **タイトルの公式:**
> 「**[対象]向け**[技術]で[成果]する**[手段]**」
> 例: 「**EC事業者向け**Python で**月20時間削減**する**自動化術**」

## 4. YouTube(=「動画でリーチ拡大」参考)(20分)

### 4-1. 戦略

- **技術解説 + 案件こなしてる風景** → YouTuber化で月+50万
- 1本3〜10分、週1〜2本

### 4-2. ジャンル

- **コーディング実況**(クロード使い方/案件解決)
- **キャリア論**(月100万までの道のり)
- **比較レビュー**(ツール/フレームワーク)
- **解説**(技術トピックを噛み砕く)

> **コツ:** 完璧主義は禁止。**「とりあえず録画して投稿」**を100本続けると風景が変わる。

## 7. メンター業(=「教えるだけで月10万」)(15分)

### 7-1. プラットフォーム

- **MENTA**(月額契約型・サブスク向き)
- **TimeTicket**(単発相談1時間〜)
- **ストアカ**(オンライン講座)

### 7-2. 単価

- 1時間 5,000〜10,000円(初心者向け)
- 月額メンター 月20,000〜50,000円(継続)
- 講座販売 1本5,000〜30,000円(**ストック収入**)

> **パッシブ化のコツ:**
> 1度作った講座を**販売し続ける**=働かなくても収入。
> 質問対応はメンバーシップ($5/月など)で限定。

| Twitter フォロワー | 月+200 |
