# Day 59:メンタル・継続術 - 365日続ける技術

## 💡 これは何?(小学生でもわかる説明)

**「365日続けるための具体的な方法」**を学ぶ日。

### なぜ大事?
- フリーランスで成功できない人の99%は**技術不足じゃなく、続けられないから**
- 「最初の1ヶ月だけ頑張る」人は山ほどいる→だから差がつく
- 続ければ**才能の差は消える**(継続>才能>努力量)
- 燃え尽き・比較病・モチベ低下に**事前に備えてる人だけ**残る

### イメージ
- ダイエット = 知識じゃなく**続けられるかどうか**で結果が決まる
- 学習・案件 = **同じ**。技術の話じゃなくて**続ける仕組み**の話
- 1日3時間 × 365日 = **約1,100時間** = プロレベルに必達
- 1日5時間 × 30日 = 150時間 = 入門レベルで終わる

> **もう1つの例え:** 
> マラソン=**ペース配分しない人ほど早く脱落**する。
> フリーランスも同じ=**飛ばしすぎ・休まなさすぎ**で潰れる。

## 1. 燃え尽き対策(=「飛ばしすぎを防ぐ」)(20分)

### 1-1. NG パターン(やったら潰れる)

- 月150万稼げる時に**働きすぎ → 体壊す**
- 「もっと、もっと」で**家族を犠牲**にする
- 比較病(SNSで他人と)で**自己否定**ループ
- 達成しても**休めずに次へ**=エネルギー枯渇

> **なぜ?**
> 人間の脳は「短期高負荷」より「長期コンスタント」が**継続できる**。
> マラソン選手はラスト1km全力ダッシュしない。

### 1-2. OK パターン(続けられる人の習慣)

- **週1日完全オフ**(コードもSNSも見ない)
- 月収目標達成したら**働く時間を減らす**(週30h上限)
- **健康診断 / 歯科** 定期検査
- **運動週3日**(散歩でOK)
- 達成したら**自分に小さなご褒美**(マッサージ・好きな食事)

### 1-3. 危険信号チェック

| サイン | 対処 |
|---|---|
| 朝起きるのが辛い | **完全休養1日**取る |
| コードを見るとうんざり | **2〜3日離れる** |
| 些細なことでイラつく | **運動・散歩** |
| 寝つきが悪い | **スマホを別部屋に** |
| 食欲がない | **病院** |

→ **早めの対処**がベスト。我慢して続けると**3ヶ月離脱**になる。

## 2. 習慣化の科学(=「やる気に頼らない仕組み」)(20分)

### 2-1. 小さく始める(超重要)


→ **ハードルを馬鹿げたほど低く**設定。

> **なぜ?**

> 「PC開いた」だけで**5分やる→気分乗ったら1時間**にスムーズ移行。

### 2-2. トリガー設計(=「既存習慣にくっつける」)

```
昼食後           → 案件チェック5分
```

→ **既存行動とセット**で続く。

### 2-3. 21日ルール(神話に注意)

- 「21日で習慣化」は**神話**(個人差大きい)
- 実際は**66日平均**(行動科学の研究結果)
- でも**3日続いたら**3週間やれる
- **30日続いたら**3ヶ月やれる
- **3ヶ月続いたら**1年やれる

> **節目で自分を褒める。**
> 「3日続いた!」「1週間できた!」=**ご褒美**で記憶に残す。

## 4. モチベ低下の対処(=「やる気でない時の処方箋」)(20分)

### 4-1. 「やる気でない」時の5ステップ

1. **15分だけやる**(始められれば乗ることが多い)
2. **超簡単な作業**から(README整理・ファイル名修正など)
3. **環境変える**(カフェ・図書館・コワーキング)
4. **散歩**してから戻る(15分歩くだけで脳がリセット)
5. それでも無理なら**休む**(罪悪感ゼロで)

> **重要:**
> 5を選んだ時に**罪悪感を持たない**。
> 「今日休んだ分、明日やればいい」=長期的にプラス。

### 4-2. 比較病の処方箋

- SNSで「年収3000万」「20代CTO」見て凹む → 普通の人類の反応
- → **3年前の自分と比較**(過去の自分一択)
- フォローを**整理**(妬み源を切る)
- **数値で自分を見る**(フォロワー・収入・案件数の伸び)

> **比較する相手の選び方:**
> ❌ トップ1%の有名人(現実離れ)
> ✅ **3年前の自分**(確実に成長してる)
> ✅ **同期で頑張ってる人**(励まし合える)

### 4-3. 失敗との付き合い方

| ありがちな失敗 | 真実 |
|---|---|
| ★1評価ついた | 1件だけ。残り99%は★5。 |
| 案件落ちた | **平均30件中1件しか採用されない**(普通) |
| バグ出した | 修正して納品 → 信頼さらに上がる |
| 質問の答え分からない | クロードに聞けば99%解決 |
| クライアント怒らせた | **冷静に謝罪→改善案提示**で半数は復活 |

→ **失敗=成長**と再定義。

## 7. 環境設計(=「ルーティンで自動運転」)(15分)

### 7-1. 朝のルーティン例

```
6:00  起床
6:05  水・軽食(バナナ等)
6:10  散歩 30分(光を浴びる=セロトニン活性)
6:40  シャワー
7:00  Deep Work 開始(Day学習 or 案件)
9:00  メール/Slack確認(ここまでスマホ見ない)
```

→ **朝に学習終わらせる**と1日が楽になる。

### 7-2. 夜のルーティン例

```
21:00 仕事終了(Slack通知OFF)
21:30 progress.md 記入
22:00 風呂(湯船で体温上げる=寝つき良)
22:30 読書 / 動画(画面の光を弱く)
23:00 就寝
```

> **コツ:**
> 寝る前1時間は**スマホを別部屋**に。
> ブルーライト=メラトニン抑制=寝付き悪化。

### 7-3. 週のルーティン

| 曜日 | 内容 |
|---|---|
| 月〜金 | 仕事・学習(週30〜40h) |
| 土 | 半日仕事 + ストック投稿作成 |
| **日** | **完全オフ**(コードもSNSも見ない) |

→ **週1完全オフ厳守**=長期的に勝つ。
