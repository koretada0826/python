# クロード依頼テンプレ集(コピペで使える)

## 基本ルール

困ったら、まず**この5つのテンプレ**で90%解決する。

---

## 1. 何かを実装したい

```
以下を実装するPythonコードを書いて。

やりたいこと:
<日本語で説明>

入力:
<どんなデータが入ってくるか>

出力:
<何を返してほしいか>

制約:
<エラー処理・パフォーマンスなど>

例:
入力: "りんご 100, みかん 200" のようなテキスト
出力: 商品ごとの合計金額の辞書
```

---

## 2. エラーが出た

```
このエラーが出た。直して。

実行したコード:
<コード全部コピペ>

エラー文:
<エラー全文コピペ、Tracebackから最後まで>

期待した動作:
<こうなってほしかった>
```

---

## 3. コードの意味が分からない

```
以下のコードを、初心者にもわかるように1行ずつ説明して。

<コードコピペ>

特に分からないところ: <あれば>
```

---

## 4. 既存コードを改善したい

```
このコードを以下の観点で改善して。

<コードコピペ>

改善観点:
- 速度
- 読みやすさ
- エラー処理
- ライブラリの選択
```

## 5. 仕様を考えるところから

```
〇〇という業務を自動化したい。

現状の手作業:
1. ...
2. ...
3. ...

時間: 〇時間/月

理想:
- 〇〇を自動化
- 〇〇を自動化

質問:
1. どんな技術スタックがおすすめ?
2. 想定される難所は?
3. 工数の目安は?

```

---

## 6. ランサーズの提案文を書く

```
以下のランサーズ案件への提案文を書いて。

【案件本文】
<コピペ>

【私の状況】
- スキル: <Python, pandas, AI連携など>
- ポートフォリオ: <URL>
- 過去実績: <あれば>
- 単価希望: <〇〇円>
- 期間: <△日>

【トーン】
丁寧 + プロフェッショナル + 信頼感
800-1200字程度
```

---

## 7. 進捗報告メールを書く

```
ランサーズの進捗報告メールを書いて。

【完了したこと】
- ...
- ...

【残りのタスク】
- ...

【予定】
〇月〇日に納品予定

トーン: 簡潔 + 信頼感
300字程度
```

---

## 8. 納品メッセージを書く

```
ランサーズの納品メッセージを書いて。

【成果物】
- main.py
- README.md
- requirements.txt
- sample_output/

【動作確認】
- ✅ 〇〇のテストパス
- ✅ ✕✕のテストパス

【使い方の要点】
- ...

クライアント名: 〇〇様

トーン: 丁寧 + 詳細 + 修正対応の意思を示す
```

---

## 9. README を書く

```
以下のプロジェクトのREADME.mdをマークダウンで書いて。

プロジェクト名: <名前>

機能:
- ...
- ...

使い方:
1. インストール
2. 実行
3. 結果

サンプルスクショ: docs/screenshot.png

含める:
- 概要(1-2行)
- 機能一覧
- 使い方
- 必要環境
- ライセンス
- 連絡先
```

---

## 10. プロフィール文を書く

```
ランサーズ・クラウドワークスのプロフィール文を書いて。

【特徴】
- Pythonでの業務自動化が得意
- データ分析・スクレイピング経験あり
- ClaudeAPI/OpenAI API でAIアプリ構築可能
- ポートフォリオあり: <GitHub URL>

【専門領域】
<EC・士業・医療など、絞れるなら>

【対応時間】
平日 19:00-23:00、土日終日

トーン: 丁寧 + 具体的 + 信頼感
1500字以内
```

---

## 11. 価格交渉

```
クライアントから値引き要求が来た。返信文を書いて。

【相手の要望】
<コピペ>

【私の希望】
- 当初提示価格: 〇〇円
- 譲歩可能ライン: △△円
- これ以下は受けたくない

【方針】
価値を再提示しつつ、譲歩できる範囲で柔軟に対応。
ノーは丁寧に。
```

---

## 12. クライアント候補へのDM

```
中小企業への直接営業DMを書いて。

【相手】
〇〇株式会社
業種: <業界>
推測される困りごと: <DXに困ってそう、業務多そう、など>

【私のサービス】
- Python自動化
- ECデータ管理が得意
- 月額契約も可能

【目標】
無料相談につなげる

トーン: 失礼にならない、専門家として、相手目線
500-800字
```

---

## 13. データ分析のコメント

```
以下のデータから、ビジネス的に重要な気づきを箇条書きで3つ挙げて。

<データ・統計値>

含める観点:
- 異常値・外れ値
- トレンド・パターン
- 改善余地
- アクション提案

トーン: 経営者向け、結論ファースト、200字以内/項目
```

---

## 14. 仕様確認

```
クライアントに送る仕様確認のメッセージを書いて。

【依頼内容】
<案件文>

【確認したいこと】
- データ件数の想定
- 出力形式の希望
- 例外時の挙動(エラー or スキップ)
- 納期の優先度

トーン: 丁寧、相手の負担にならない、Yes/Noで答えやすい
```

---

## 15. 学習質問

```
以下について教えて。小学生でもわかるように。

質問: <例: pandas の groupby って何?>

求める形式:
- 最初に1行で結論
- 簡単な例コード
- いつ使うか
- 注意点
```

## 困ったら

このテンプレで足りなければ、**自分の言葉でクロードに聞く**だけでOK。
クロードは賢いので、文章が下手でも理解してくれる。

→ **悩むより質問**。
